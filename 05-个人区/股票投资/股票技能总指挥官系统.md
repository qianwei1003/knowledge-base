# 股票技能总指挥官系统 v1.0

**设计理念**: 像一个专业的投资顾问团队Leader，根据用户需求自动调度最合适的技能组合

---

## 🎯 核心架构

```
用户提问
    ↓
[意图识别层] → 分析用户需求类型
    ↓
[技能调度层] → 选择最优技能组合
    ↓
[执行层] → 调用技能获取结果
    ↓
[整合层] → 综合输出+操作建议
    ↓
用户获得：数据/分析/决策建议
```

---

## 📊 五层技能体系

| 层级 | 技能名称 | 核心功能 | 响应时间 | 使用频率 |
|------|----------|----------|----------|----------|
| **L1 数据层** | neodata-financial-search | 实时数据查询 | 2-3秒 | ⭐⭐⭐⭐⭐ 每天 |
| **L2 分析层** | trading-analysis | 深度投资分析 | 30-60秒 | ⭐⭐⭐⭐ 调仓时 |

| **L4 风控层** | risk-metrics-calculation | 风险指标计算 | 5-10秒 | ⭐⭐⭐ 建仓前 |
| **L5 验证层** | backtesting-frameworks | 策略回测框架 | 手动 | ⭐⭐ 开发策略时 |

## 🎭 用户意图识别矩阵

### 场景1: "这只股票怎么样？"
**意图**: 快速了解个股基本情况  
**调度方案**:
```

├─ 实时行情（价格/涨跌幅/市值）
├─ 基本面（PE/PB/ROE）
├─ 资金流向（主力净流入）
└─ 近期新闻（催化剂/风险）

输出格式: 一页纸摘要
```

**用户提问技巧**:
| ❌ 低效提问 | ✅ 高效提问 |
|------------|------------|

```
├─ 信心水平: 高/中/低
├─ 建议仓位: %
├─ 入场价位: XX-XX元
├─ 目标价位: XX元
├─ 止损价位: XX元
└─ 核心理由: 3-5句话
```

**用户提问技巧**:

**意图**: 风险评估  
**调度方案**:
```

```

## 💡 用户沟通最佳实践

### DOs (这样做)
1. **明确标的**: "300750" 比 "宁德时代" 更精准
2. **说明时间**: "近一季度" 比 "最近" 更具体
3. **指定维度**: "估值和资金流" 比 "怎么样" 更明确
4. **给出场景**: "能不能买入" 会触发分析模式
5. **提供背景**: "已持有" vs "想建仓" 建议不同

### DON'Ts (避免这样)
1. ❌ 过于笼统: "这个股好吗"
2. ❌ 缺乏标的: "今天有什么新闻"
3. ❌ 模糊时间: "之前的表现"
4. ❌ 多重问题: "价格、财报、估值、能不能买"
5. ❌ 代词指代: "它怎么样"（指代不清）

## 🎓 沟通技巧进阶

### Level 1: 基础查询 (每天使用)
```
"[股票代码] 最新[数据类型]"

示例:
- "300750 最新股价"
- "宁德时代 市盈率"
- "000858 主力资金流向"
```

### Level 2: 对比分析 (研究时使用)
```
"对比 [股票A] 和 [股票B] 的 [指标]"

示例:
- "对比宁德时代和比亚迪的估值"
- "中国西电和思源电气哪个更值得买"
- "300750和000858的ROE对比"
```

### Level 3: 组合分析 (投资决策)
```
"分析 [标的] 的 [场景] 投资价值"

示例:
- "分析一下300750能不能买入"
- "评估持有宁德时代的风险"
- "如果买入10万宁德时代，最坏亏多少"
```

### Level 4: 系统选股 (发现机会)
```
"筛选 [条件] 的股票"

示例:
- "帮我找PE低于20倍的价值股"
- "筛选电力设备板块的高增长股票"
- "找找近期有催化剂的新能源股"
```

### Level 5: 策略验证 (量化研究)
```
"回测 [策略] 在 [标的] 上的表现"

示例:
- "回测MA20上穿MA60买入策略"
- "测试双均线策略在宁德时代的效果"
- "验证突破买入策略的胜率"
```

## 📝 总结：与AI协作的三种模式

| 模式 | 适用场景 | 用户角色 | AI角色 |
|------|----------|----------|--------|
| **查询模式** | 快速查数据 | 提问者 | 数据助手 |
| **分析模式** | 投资决策 | 决策者 | 投资顾问 |
| **探索模式** | 选股/发现 | 探索者 | 研究助手 |
| **验证模式** | 策略测试 | 研究员 | 量化工程师 |

**核心原则**: 
- 越具体的提问 → 越精准的回答
- 提供越多背景 → 获得越相关建议
- 明确使用场景 → 触发最优技能组合
