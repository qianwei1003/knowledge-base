## 修复详情

### 修复文件: `core/data_fetcher.py`

**问题**: akshare 网络连接失败，且无法正确回退到 neodata

**解决方案**:

1. **修复脚本路径**（第41-47行）：
```python
# 修复前（相对路径容易出错）
NEODATA_SCRIPT = os.path.join(
    os.path.dirname(os.path.dirname(os.path.abspath(__file__))),
    "..", "..", "..",
    ".codebuddy", ...
)

# 修复后（绝对路径稳定）
NEODATA_SCRIPT = r"C:\Users\60597\.codebuddy\plugins\marketplaces\cb_teams_marketplace\plugins\finance-data\skills\neodata-financial-search\scripts\query.py"
```

2. **改进错误处理**（第89-106行）：
```python
# 修复前：捕获不到网络异常
except DataFetchError as e:
    print(f"akshare 获取失败: {e}")

# 修复后：捕获所有异常
except DataFetchError as e:
    print(f"akshare 获取失败: {e}，将尝试 neodata...")
except Exception as e:
    print(f"akshare 异常: {e}，将尝试 neodata...")
```

## 已知限制

1. **akshare 网络问题**：由于代理设置，akshare 无法直接连接东方财富
   - ✅ 已解决：自动回退到 neodata

2. **trend 维度数据不足**：neodata 返回的数据缺少完整历史数据
   - 影响：趋势评分可能为0或偏低
   - 建议：结合 trading-analysis 进行深度分析

3. **响应速度**：neodata 网络请求约 2-3 秒/次
   - 单股分析：约 5-8 秒
   - 板块扫描：约 1-2 分钟（取决于股票数量）
