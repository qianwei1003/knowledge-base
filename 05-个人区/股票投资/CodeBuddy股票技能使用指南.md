## 二、核心技能详解

### 🔥 第一优先级：最常用的 5 个技能

#### 1. neodata-financial-search（自然语言金融搜索）

| 属性 | 说明 |
|------|------|
| **能力强度** | ⭐⭐⭐⭐⭐（必须掌握） |
| **响应速度** | 快（实时数据） |
| **适用市场** | A股/港股/美股/基金/宏观/外汇/大宗商品 |

**使用场景**：
- 快速查询股票行情、财报、资金流向
- 自然语言提问，无需记API

**触发词示例**：
```
"宁德时代最新股价多少？"
"贵州茅台今天涨了多少？"
"光芯片板块资金流向如何？"
"比亚迪最新财报，营收和净利润"
"最近一周涨停的股票有哪些？"
```

**实际效果**：
- ✅ 实时行情数据
- ✅ 财务报表（利润表/资产负债表/现金流量表）
- ✅ 板块资金流向、热度排名
- ✅ 新闻、公告、研报
- ❌ 分钟级历史数据（需用 finance-data-retrieval）

---

#### 2. trading-analysis（多角色投资分析）

| 属性 | 说明 |
|------|------|
| **能力强度** | ⭐⭐⭐⭐⭐（专业级） |
| **响应速度** | 慢（多 Agent 协作，约 2-5 分钟） |
| **输出质量** | 机构级投资分析报告 |

**使用场景**：
- 重大投资决策前深度分析
- 获取专业的 BUY/SELL/HOLD 建议

**触发词示例**：
```
"分析一下宁德时代能不能买"
"给我一份腾讯控股的投资分析报告"
"中国平安值得投资吗？深度分析"
"帮我分析这个股票：代码 300750"
```

**输出内容**：
- 技术面分析（趋势、支撑压力位、动量）
- 基本面分析（盈利能力、成长性、估值）
- 新闻面分析（重要事件、宏观影响）
- 情绪面分析（资金流向、机构观点）
- 投资辩论（多头 vs 空头观点）
- **最终决策**：BUY/SELL/HOLD + 仓位建议 + 目标价/止损价

**使用建议**：
- 适合 10 万元以上级别的投资决策
- 不适合高频快速决策（速度较慢）
- 建议每天最多使用 3-5 次（避免过度分析）

---

#### 3. idea-generation（选股/发现机会）

| 属性 | 说明 |
|------|------|
| **能力强度** | ⭐⭐⭐⭐（实用） |
| **响应速度** | 中等 |
| **适用场景** | 不知道买什么时 |

**使用场景**：
- 寻找投资机会
- 系统化筛选股票

**触发词示例**：
```
"帮我筛选一些价值股"
"找找 AI 板块的投资机会"
"推荐一些有潜力的成长股"
"看看有什么被低估的股票"
```

**筛选策略**：
- 价值筛选：低 PE、高股息、自由现金流 >5%
- 成长筛选：营收增长 >15%、盈利增长 >20%
- 质量筛选：ROE>15%、稳定毛利率、低负债
- 做空筛选：营收下滑、毛利压缩、内部人抛售

**输出**：5-10 个 idea，每个包含：
- 公司概况
- 关键财务指标
- 投资逻辑（3-5 点）
- 风险因素
- 下一步建议

---

#### 4. backtesting-frameworks（策略回测）

| 属性 | 说明 |
|------|------|
| **能力强度** | ⭐⭐⭐⭐⭐（专业量化级） |
| **响应速度** | 取决于数据量 |
| **适用场景** | 验证策略有效性 |

**使用场景**：
- 验证买入信号策略是否有效
- 测试技术指标组合
- 避免"感觉有效"的陷阱

**触发词示例**：
```
"帮我回测一下这个策略：MA20上穿MA60买入"
"测试一下突破策略在过去3年的表现"
"回测均线策略在宁德时代上的表现"
```

**回测框架包含**：
- 事件驱动回测（最准确）
- 向量化回测（最快）
- Walk-Forward 分析（防过拟合）
- 蒙特卡洛模拟（风险评估）

**关键指标**：
- 总收益率、年化收益率
- 最大回撤
- 夏普比率
- 胜率

**使用建议**：
- 任何策略实盘前必须回测
- 至少回测 3 年以上数据
- 关注最大回撤，而非只看收益

---

#### 5. risk-metrics-calculation（风险计算）

| 属性 | 说明 |
|------|------|
| **能力强度** | ⭐⭐⭐⭐（专业级） |
| **响应速度** | 快 |
| **适用场景** | 评估组合风险 |

**使用场景**：
- 计算持仓风险
- 设定止损位
- 评估最大可能损失

**触发词示例**：
```
"计算一下我这个持仓组合的风险"
"算算这只股票的VaR"
"评估一下最大回撤风险"
```

**风险指标**：
- VaR（在险价值）：95%置信度下最大损失
- CVaR（条件在险价值）：极端情况平均损失
- 最大回撤：历史上最大亏损幅度
- 夏普比率：风险调整后收益
- 压力测试：2008危机/2020疫情等场景表现

## 三、投资决策流程指南

### 场景 1："我想买一只股票，但不知道能不能买"

**推荐流程**：
```
步骤 1: 使用 neodata-financial-search 查询基本信息
        "宁德时代最新股价和基本面"
        
步骤 2: 使用 trading-analysis 进行深度分析
        "深度分析宁德时代，能不能买入？"
        
步骤 3: 根据报告决策
        ├── 建议 BUY + 信心高 → 可以考虑买入
        ├── 建议 HOLD → 观望
        └── 建议 SELL → 不建议买入
```

**时间成本**：5-10 分钟
**资金门槛**：适合 5 万元以上决策

---

### 场景 2："我不知道买什么，想寻找机会"

**推荐流程**：
```
步骤 1: 使用 idea-generation 筛选
        "帮我找一些被低估的价值股"
        
步骤 2: 筛选出 3-5 个候选

步骤 3: 对每个候选使用 trading-analysis 深度分析
        "分析 [股票A] 的投资价值"
        "分析 [股票B] 的投资价值"
        
步骤 4: 综合比较，选择 1-2 个买入
```

**时间成本**：30-60 分钟
**产出**：1-2 个高质量投资机会

---

### 场景 3："我有一个策略想法，想验证是否有效"

**推荐流程**：
```
步骤 1: 明确策略规则
        例："当MA20上穿MA60，且成交量放大2倍时买入"
        
步骤 2: 使用 backtesting-frameworks 回测
        "回测策略：[策略描述]"
        "测试时间：2021-01-01 到 2024-12-31"
        "标的：沪深300成分股"
        
步骤 3: 分析回测结果
        ├── 夏普比率 > 1.0 → 策略可能有效
        ├── 最大回撤 < 20% → 风险可控
        └── 胜率 > 50% → 正向期望
        
步骤 4: 使用 risk-metrics-calculation 评估风险
        "计算这个策略的VaR和压力测试"
```

**时间成本**：1-2 小时
**产出**：策略有效性结论 + 风险参数

---

### 场景 4："我想建立一套买入信号系统"

**推荐流程**（结合你原有的技术方案）：
```
阶段 1: 数据获取
        ├── neodata-financial-search：实时数据
        ├── finance-data-retrieval：历史分钟线
        └── 数据整合
        
阶段 2: 信号设计
        ├── 技术面：趋势、形态、异动（40%权重）
        ├── 情绪面：资金流向、板块热度（35%权重）
        └── 基本面：财务健康、新闻（25%权重）
        
阶段 3: 回测验证
        └── backtesting-frameworks 验证信号有效性
        
阶段 4: 风险管理
        └── risk-metrics-calculation 设定止损/仓位
        
阶段 5: 实盘监控
        └── catalyst-calendar 追踪重要事件
```

## 五、常见误区与避坑指南

### ❌ 误区 1：过度依赖 trading-analysis

**问题**：每只股票都用 trading-analysis，耗时过长
**解决**：
- 先用 neodata-financial-search 快速筛选
- 只对有初步兴趣的股票用 trading-analysis
- 建议每日最多 3-5 次深度分析

### ❌ 误区 2：忽略回测直接实盘

**问题**：感觉策略有效就直接用钱测试
**解决**：
- 任何策略必须先用 backtesting-frameworks 回测
- 至少回测 3 年数据
- 关注最大回撤，而非只看收益率

### ❌ 误区 3：不设置止损

**问题**：买入后不设止损，亏损扩大
**解决**：
- 使用 risk-metrics-calculation 计算合理止损位
- 单笔交易最大亏损不超过总资金 2%
- 使用 trailing stop（移动止损）保护利润

### ❌ 误区 4：频繁交易

**问题**：每天多次交易，被手续费和滑点侵蚀
**解决**：
- 用 catalyst-calendar 规划交易节奏
- 只在高置信度信号出现时交易
- 降低交易频率，提高单笔质量

## 六、实战示例

### 示例 1：完整的买入决策流程

**用户需求**："我想买入宁德时代，请帮我分析"

**执行流程**：

1️⃣ **快速数据获取**（30秒）
```
用户: "宁德时代最新股价和基本面"
AI: 股价 185.5 元，市值 8152 亿，PE 25.3x，PB 4.8x
   近一年涨幅 +12%，近一月涨幅 -5%
   Q3 营收 920 亿，同比 +25%，净利润 95 亿，同比+28%
```

2️⃣ **深度分析**（3-5分钟）
```
用户: "深度分析宁德时代，能不能买入？"
AI: [触发 trading-analysis]
   
   === 投资分析报告 ===
   
   最终建议: BUY（买入）
   信心水平: 中高
   风险等级: 中等
   建议持仓期: 中期（1-6月）
   建议仓位: 5-8%
   
   核心理由:
   1. 基本面：Q3 业绩超预期，毛利率回升至 22.3%
   2. 技术面：回调至支撑位，RSI 超卖区域
   3. 情绪面：北向资金连续 5 日净流入
   4. 估值：PE 25x 处于历史 30% 分位，相对合理
   
   操作建议:
   - 入场价位: 180-190 元
   - 目标价位: 220-230 元（+20%）
   - 止损价位: 165 元（-10%）
   - 仓位比例: 5-8%
   - 操作节奏: 分批建仓（先 3%，确认趋势后加 2-3%）
```

3️⃣ **决策**
```
用户根据报告决定是否买入
```

---

### 示例 2：策略回测验证

**用户需求**："我想测试 MA20 上穿 MA60 的买入策略"

**执行流程**：

1️⃣ **回测**（2-3分钟）
```
用户: "回测策略：当MA20上穿MA60且成交量放大时买入，持有20天"
        "回测标的：宁德时代，时间：2021-01到2024-12"
AI: [触发 backtesting-frameworks]

   === 回测结果 ===
   
   总收益率: +45.2%
   年化收益率: +12.8%
   最大回撤: -18.5%
   夏普比率: 0.92
   胜率: 52%
   盈亏比: 1.8
   交易次数: 38次
   
   评估: 策略有一定效果，但夏普比率略低于 1.0，建议优化
```

2️⃣ **风险计算**（30秒）
```
用户: "计算这个策略的VaR"
AI: VaR(95%): -2.3%（单笔交易最大损失）
   CVaR: -3.1%（极端情况平均损失）
   建议单笔仓位不超过 5%
```

## 八、总结

### 核心要点

1. **数据查询**：`neodata-financial-search` 是入口，随时可用
2. **深度分析**：`trading-analysis` 是核心，用于重大决策
3. **策略验证**：`backtesting-frameworks` 是保障，避免凭感觉
4. **风险管理**：`risk-metrics-calculation` 是底线，保护本金

### 建议学习顺序

```
第1周: 熟练使用 neodata-financial-search
       练习各种查询：行情、财报、资金流向

第2周: 使用 trading-analysis 分析 5-10 只股票
       理解报告结构和决策逻辑

第3周: 使用 idea-generation 寻找机会
       建立自己的选股流程

第4周: 学习 backtesting-frameworks
       回测 2-3 个简单策略

第5周: 整合所有技能
       建立完整的投资决策流程
```
