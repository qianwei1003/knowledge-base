## 一、核心原则（修订版）

> **⚠️ 强制执行声明**：以下规则**不可绕过**，违反即拒绝输出！

### 强制检查清单（7项）
| # | 检查项 | 不合格处理 |
|---|--------|-----------|
| 1 | 数据时效性 | 拒绝输出，无数据截止时间 |
| 2 | 逻辑自洽 | 拒绝输出，同一标的多种建议 |
| 3 | 估值依据 | 拒绝输出，无任何估值指标 |
| 4 | 风险提示 | 拒绝输出，无止损建议 |
| 5 | 仓位合规 | 拒绝输出，未考虑用户现有持仓 |
| 6 | 语言严谨 | 拒绝输出，绝对化表述 |
| 7 | 知识库更新 | 完成前不输出 |

### 基础原则

1. **数据时效性优先**
   - 所有分析必须标注**数据截止日期**
   - 明确说明周末/节假日休市导致的时效延迟
   - 禁止使用过夜数据（超过T+1）进行"下周预测"

2. **逻辑自洽性**
   - 每只股票**只能出现在一个类别**中（买入/持有/卖出）
   - 同一标的的正负面因素应整合为综合评价
   - 避免同一标的既推荐买入又提示卖出的矛盾

3. **专业严谨性**
   - 使用复合估值指标（PE/PB/PS/行业对比）
   - 止损位应有明确依据（技术支撑/均线/历史高低点）
   - 避免绝对化表述（如"最强"、"必涨"）

4. **风险提示完整性**
   - 每次分析必须包含风险提示
   - 明确标注仓位建议和止损比例

## 五、逻辑规范（重要）

### 5.1 避免自相矛盾

```markdown
❌ 错误示例：
- 长光华芯 → 建议买入（资金流入）
- 长光华芯 → 建议卖出（PE过高）
[同一标的同时出现两个相反建议]

✅ 正确做法：
- 长光华芯 → 建议买入
  - 优势：资金持续流入，行业景气度高
  - 风险：PE估值偏高，短期波动大
  - 操作：控制仓位，设定止损
```

### 5.2 单一类别原则

```
每只股票只能出现在一个分类中：
- ✅ 建议买入：明确推荐做多
- ✅ 谨慎持有：中性评价，不建议新增
- ✅ 建议卖出/回避：明确提示风险
```

### 5.3 避免的表述

| 禁止表述 | 替代表述 |
|----------|----------|
| "下周必涨/必跌" | "基于近期表现，建议关注" |
| "短期最强" | "近期表现强势" |
| "追高买入" | "注意追高风险，可考虑回调后介入" |
| "逢低买入" | "建议在支撑位附近分批建仓" |

## 六、质量检查清单

> **强制声明**：每次投顾输出前必须执行7项检查，**全部通过方可输出**

### 快速检查（7项）

```
□ 1. 数据时效性：数据截止时间明确标注
□ 2. 逻辑一致性：同一标的只有一种建议
□ 3. 估值依据：至少1个估值指标（PE/PB/PS）
□ 4. 风险提示：包含止损建议和仓位控制
□ 5. 仓位合规：考虑用户现有持仓情况
□ 6. 语言严谨：无绝对化表述（必涨/必跌/最强）
□ 7. 知识库同步：分析后更新知识库记录
```

### 不合格处理

| 检查项失败 | 处理方式 |
|-----------|---------|
| 数据时效性 | 拒绝输出，要求补充数据 |
| 逻辑一致性 | 拒绝输出，合并矛盾观点 |
| 估值依据 | 拒绝输出，补充估值分析 |
| 风险提示 | 拒绝输出，添加风险说明 |
| 仓位合规 | 拒绝输出，先澄清仓位 |
| 语言严谨 | 拒绝输出，修改措辞 |
| 知识库同步 | 暂停输出，完成更新 |
