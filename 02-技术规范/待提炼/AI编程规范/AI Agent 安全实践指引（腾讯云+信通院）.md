# AI Agent 安全实践指引

> 发布机构：腾讯云计算（北京）有限责任公司、中国信息通信研究院人工智能研究所
> 数据来源：OWASP风险框架、工信部NVDB平台、ClawHub技能市场（2026年2月数据）

## 一、背景与核心痛点

随着 AI Agent 在企业规模化部署，其自主决策与工具调用能力挑战了传统安全体系，引发权限滥用、供应链风险等新型威胁。

2026年2月研究显示，技能市场恶意组件比例达 **12%（341/2857）**（ClawHub审计数据），亟需系统性安全指南。

### 核心痛点

1. **权限溢出风险**：AI Agent 默认管理员权限可致数据误删、系统篡改（OWASP定义）
2. **供应链攻击**：12% 第三方技能包含恶意代码（ClawHub审计），可窃取会话上下文
3. **审计缺位**：跨模型、工具、插件的决策链无日志追踪，根因分析难度提升 300%

## 二、AI Agent 五类高发风险

### 2.1 权限管控不当 — "小问题变成大事故"
AI Agent 拥有过高权限时，一个误操作或被注入的指令可能导致：
- 数据误删
- 系统配置篡改
- 敏感信息泄露

### 2.2 外部组件存隐患 — 引入供应链风险
- 第三方 Skills/Plugins 可能包含恶意代码
- 恶意组件可窃取会话上下文、注入提示词
- 12% 的第三方技能包存在安全问题

### 2.3 输入内容不设防 — 引发敏感信息泄露
- 用户输入中可能包含提示注入攻击
- 敏感数据（API Key、密码）可能被 Agent 意外转发
- 缺乏输入过滤和脱敏机制

### 2.4 运行环境隔离不足 — 造成风险横向扩散
- Agent 进程与宿主系统未隔离
- 一个 Agent 被攻破可影响同环境其他服务
- 缺乏沙箱和容器化隔离

### 2.5 审计溯源机制缺位 — "出了事也说不清"
- 跨模型、工具、插件的决策链无日志追踪
- 根因分析难度极高
- 无法回溯 Agent 的具体操作路径

## 三、安全使用原则：「六要六不要」

### 3.1 版本控制原则
✅ **要**：对所有 Agent 配置、Skills、Rules 进行版本管理
❌ **不要**：随意修改未经审查的 Agent 配置

### 3.2 暴露面管控原则
✅ **要**：最小化 Agent 对外暴露的接口和端口
❌ **不要**：将 Agent 管理界面直接暴露在公网

### 3.3 最小权限原则
✅ **要**：Agent 只授予完成任务所需的最低权限
❌ **不要**：给 Agent 默认管理员或 root 权限

### 3.4 组件使用原则
✅ **要**：建立 Skills 白名单机制，只使用经过审计的组件
❌ **不要**：直接安装未经验证的第三方技能包

### 3.5 攻击防范原则
✅ **要**：对用户输入进行过滤和脱敏，防止提示注入
❌ **不要**：将用户原始输入直接传递给模型执行

### 3.6 持续防护原则
✅ **要**：建立持续监控和审计机制，记录所有 Agent 操作
❌ **不要**：部署后不再关注 Agent 的运行状态

## 四、三层防护体系：「部署-运行-保障」

### 4.1 部署层：基础加固到企业级控制
- **权限动态围栏**：高风险操作需人工二次确认
- **组件安全核验**：扫描恶意代码与提示词注入
- **环境隔离**：使用沙箱/容器隔离 Agent 运行环境

### 4.2 运行层：输入到执行的三道防线
1. **第一道防线 — 输入过滤**：检测并过滤提示注入攻击
2. **第二道防线 — 权限校验**：动态检查操作权限，敏感操作需审批
3. **第三道防线 — 执行监控**：实时监控 Agent 行为，异常操作自动熔断

### 4.3 保障层：五层全链条监测防护
1. **资产盘点**：梳理所有 AI Agent 资产和权限
2. **Token 限流**：防止资源滥用和费用失控
3. **数据防泄漏**：敏感数据脱敏和访问控制
4. **行为审计**：记录 JSONL 格式日志，100% 可追溯
5. **威胁情报**：实时同步恶意组件和攻击手法

## 六、总结

AI Agent 的安全防护是一个系统性工程，需要从「权限管控、供应链安全、输入过滤、环境隔离、审计溯源」五个维度全面覆盖。核心原则是：

1. **最小权限** — 宁可多确认一次，不可多授权一分
2. **白名单机制** — 只使用经过审计的组件和技能
3. **全链路审计** — 每一步操作都记录，每一个决策都可追溯
4. **持续防护** — 安全不是一次性工作，是持续运营的过程
