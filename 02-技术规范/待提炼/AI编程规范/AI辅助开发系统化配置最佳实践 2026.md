# AI 辅助开发最佳实践：2026 年新方法

> 核心理念：系统化配置代替重复指令，让 AI 一次配置、长期生效、团队共享。

## 一、为什么需要系统化的 AI 配置

### 传统方式的痛点
- 每次都要重复说同样的话 — "用TypeScript"、"注意暗色模式"、"用Tailwind"
- 好的实践没法传承 — 踩过的坑、学到的技巧，用完就忘了
- 团队配置不统一 — 每个人都要从头配置

### 系统化配置的价值
| 优势 | 说明 |
|------|------|
| 少说话多办事 | 一次配置，长期生效 |
| 团队一起用 | 配置文件提交 Git，复制到新项目就行 |
| 经验不丢失 | 踩过的坑、学到的技巧都能沉淀下来 |
| 质量更稳定 | 规范化的代码，去哪都一致 |

## 二、五大核心概念

```
┌─────────────────────────────────────────────────────────┐
│                    你的 AI 开发助手                       │
├─────────────────────────────────────────────────────────┤
│  MCP     ──→ 给AI装上"手"和"脚"，能干活                  │
│  Rules   ──→ 制定"项目宪法"，让AI知道怎么做              │
│  Skills  ──→ 召唤"领域专家"，专门回答特定问题            │
│  Agents  ──→ 分配"专人负责"，处理特定任务                │
│  Hooks   ──→ 设置"自动触发"，保存提交时执行              │
└─────────────────────────────────────────────────────────┘
```

## 三、MCP：给 AI 装上"手"和"脚"

### 3.1 MCP 是什么
MCP（模型上下文协议）就像给 AI 装上了手和脚。AI 可以：
- 📁 读取电脑上的文件
- 💻 执行终端命令
- 🗄️ 操作数据库
- 🌐 控制浏览器
- 🔧 运行各种工具

### 3.2 安装 MCP
**方式一：市场一键安装（推荐）**
1. 打开 Cursor 设置：`Cmd + ,`
2. 找到 "MCP" 或 "Extensions"
3. 点击 "Browse MCP Marketplace"
4. 搜索想要的 MCP，点击 "Add" 安装

**方式二：手动配置**
在项目根目录创建 `.cursor/mcp.json`：
```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "${workspaceFolder}"]
    }
  }
}
```

### 3.3 常用 MCP 推荐
| MCP | 用途 | 场景 |
|-----|------|------|
| Figma | 设计稿转代码 | 看设计图写代码 |
| Playwright/Puppeteer | 浏览器自动化 | E2E测试、爬虫 |
| GitHub | 代码仓库操作 | 管理PR、Issue、代码审查 |
| Browsertools | 浏览器调试 | 看控制台、网络请求 |
| Web Search | 联网搜索 | 查资料、搜文档 |
| Sequential Thinking | 逐步思考 | 复杂问题理清思路 |

## 四、Rules：制定"项目宪法"

### 4.1 Rules 是什么
Rules 告诉 AI 这个项目要怎么做，例如：
- 我们用 Next.js 的 App Router
- 默认用 Server Components
- 暗色模式用 `dark:` 前缀
- API 错误格式统一用 `{ success: boolean, error?: string }`

### 4.2 配置 Rules
**存放位置：**
- `.cursor/rules/` — 当前项目，提交 Git 后团队共享
- `~/.cursor/rules/` — 所有项目都能用

**文件格式（.mdc 文件）：**
```yaml
---
name: nextjs
description: Next.js 15 App Router项目规范
globs: ["**/*"]
---

# Next.js 15 项目规范

## 技术栈
- 框架：Next.js 15 App Router
- 语言：TypeScript
- 样式：Tailwind CSS

## 代码规范
1. 默认使用Server Components
2. 客户端用'use client'标记
3. API错误格式统一
```

**关键字段：**
| 字段 | 说明 |
|------|------|
| name | 规则名称 |
| description | 什么时候用这个规则 |
| globs | 哪些文件要遵守，如 `["**/*.tsx"]` |
| alwaysApply | true=始终生效，false=需要时才用 |

### 4.3 使用方式
- **始终生效**：配置 `alwaysApply: true`
- **智能判断**：AI 根据内容判断要不要用（默认）
- **匹配文件**：编辑匹配文件时自动应用
- **手动触发**：聊天时 `@规则名` 来引用

## 五、Skills：召唤"领域专家"

### 5.1 Skills 是什么
Skills 是"领域专家"。配置后，AI 遇到相关问题时自动用专家的思维方式来回答。

### 5.2 配置 Skills
**目录结构：**
```
.cursor/skills/
└── database/          # 一个Skill一个文件夹
    ├── SKILL.md       # 必填：技能定义
    ├── references/    # 可选：参考资料
    └── scripts/       # 可选：可执行脚本
```

**SKILL.md 格式：**
```yaml
---
name: database
description: 回答数据库相关问题时优先考虑性能
---

# 数据库专家

你是一个数据库专家，精通PostgreSQL、MySQL、Prisma。

## 回答原则
1. 优先考虑查询性能，避免N+1
2. 合理使用索引
3. 关联查询用include/preload

## 回答格式
先解释原理，再给代码示例
```

### 5.3 使用方式
- **自动触发**：AI 根据问题判断该用什么技能
- **手动触发**：输入 `/数据库` 来召唤数据库专家

## 六、Agents：分配"专人负责"

### 6.1 Agents 是什么
Agents 和 Skills 不同：Skills 回答问题，Agents 帮你干活。

### 6.2 配置 Agents
**目录结构：**
```
.cursor/agents/
└── code-review.md     # 一个Agent一个文件
```

**格式：**
```yaml
---
name: code-review
description: 代码审查专家，审查PR和代码质量
model: fast
---

# 代码审查专家

你是一个代码审查专家。请审查以下代码：
1. 潜在问题
2. 性能优化点
3. 代码规范问题
```

**model 字段：**
| 值 | 速度 | 适用场景 |
|----|------|---------|
| fast | 快 | 简单任务、代码审查 |
| balanced | 平衡 | 大多数任务 |
| smart | 慢但聪明 | 复杂任务、架构设计 |

## 七、Hooks：设置"自动触发"

### 7.1 配置 Hooks
创建 `.cursor/hooks.json`：
```json
{
  "hooks": [
    {
      "match": ".*\\.(ts|tsx|js|jsx)$",
      "run": "npx prettier --write {file}"
    },
    {
      "match": ".*\\.(ts|tsx)$",
      "run": "npx eslint --fix {file}"
    }
  ]
}
```

`{file}` 会自动替换成实际文件路径。

## 八、组合使用工作流

### 典型工作流
```
你：帮我写个用户管理模块

AI自动触发：
1. Rules → 知道项目用Next.js + TypeScript + Tailwind
2. Skills → 召唤数据库专家来处理数据层
3. MCP → 自己读取现有的用户表结构
4. Agents → 分配给代码生成Agent来写代码
5. Hooks → 写完后自动格式化
```

### 组合使用示例
```
@nextjs @skill database
帮我写一个用户管理模块，包含：
- 用户列表（分页）
- 用户CRUD
- 数据库优化
```

同时触发了 Next.js 规范 + 数据库专家技能，AI 自动用专家角度写代码。

## 九、总结

| 工具 | 作用 | 配置位置 |
|------|------|---------|
| MCP | 手和脚 | 市场安装或 `.cursor/mcp.json` |
| Rules | 项目宪法 | `.cursor/rules/` |
| Skills | 领域专家 | `.cursor/skills/` |
| Agents | 专人负责 | `.cursor/agents/` |
| Hooks | 自动触发 | `.cursor/hooks.json` |

把这五个工具组合起来，AI 开发效率会大幅提升。
