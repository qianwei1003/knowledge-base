# Cursor AI IDE 高效使用指南 - 来自 Cursor 设计负责人的 12 条关键方法

> 来源：稀土掘金 - 如何有效地使用 Cursor AI IDE 的十二个关键方法
> 作者：Cursor 设计负责人 @ryolu_

---

## 一、基础理念篇

### 1. 理解"聊天式"编程

Cursor 代表了新时代的编程方式，不再单纯地关注代码的字面构建，而是聚焦于问题的解决。通过自然语言与 AI 的互动，我们能更清晰地表达需求和想法，提升工作效率。

### 2. 熟悉 Cursor 的四大功能模块

Cursor 提供的四个模块各有特点，掌握它们可以帮助你在不同场景下灵活应用：

| 模块 | 功能 | 使用场景 |
|------|------|----------|
| **Tab** | 智能代码补全 | 快速编码时自动预测下一步 |
| **Inline Chat** | 行内对话编辑 | 选中代码后快速修改 |
| **Ask** | 问答模式 | 询问代码逻辑、解释代码 |
| **Agent** | 智能代理模式 | 通过自然语言创建、编辑和管理文件 |

### 3. 设置明确的项目规则

在开始前设置 5-10 条清晰的规则，让 Cursor 了解项目结构和限制条件：

- 使用 `/generate rules` 命令为现有代码库生成规则
- 在 `.cursor/` 目录下存放架构文档，帮助 AI 理解整体项目背景
- 明确技术栈选择（如"必须使用 ES6 语法而非 var"）

### 4. 编写具体的提示

像写小规格说明书一样描述你的需求。模糊的指令只会带来模糊的结果。

**有效模板**：技术栈 + 功能要求 + 注意事项

```
❌ 模糊：写一个登录功能
✅ 清晰：使用 React 和 TypeScript 实现 OAuth2.0 登录，
        不依赖第三方库，并支持深色模式
```

## 二、工作流程篇

### 5. 逐文件工作

将复杂任务拆分成单个文件来处理，更高效也更容易调试。

**推荐流程**：生成 → 测试 → 审查 → 下一步

举例：开发电商系统时，先从购物车组件开始，完成后再进行订单模块。

### 6. 先编写测试

用测试驱动开发（TDD），让 Cursor 不断生成直到测试通过。

**步骤**：
1. 手动编写测试用例（如 Jest）
2. 让 Cursor 生成代码
3. 反复优化直到全部通过
4. 遇到错误可将报错信息反馈给 AI

### 7. 审查 AI 输出

始终检查 AI 生成的代码，修复任何问题，并告诉 Cursor 将修复后的代码作为示例。

**方法**：修改后加上 `@fixed` 标签说明问题所在。

示例：若 AI 忘记添加 JWT 验证，修正后标注 `@fixed: 添加 JWT 验证逻辑`

## 三、进阶技巧篇

### 8. 使用范围限定

使用 `@file`、`@folders`、`@git` 等命令将 Cursor 的注意力限定在代码库的相关部分。

```
示例：
- @file:src/components/Button.tsx - 引用特定文件
- @folders:src/utils - 引用整个文件夹
- @git:HEAD~5 - 引用最近的提交
```

### 9. 保存设计文档

将设计文档和检查清单保存在 `.cursor/` 目录中，使 AI Agent 能够获取完整上下文。

```
项目结构示例：
.cursor/
├── architecture.md      # 架构文档
├── coding-standards.md  # 编码规范
├── checklist.md         # 检查清单
└── prompts/             # 常用提示模板
    ├── component-generation.md
    └── test-generation.md
```

### 10. 遇到错误直接修改

如果代码有错，直接自己修改比解释更有效，Cursor 从编辑中学习更快。

**最佳实践**：
- 先让 AI 生成初步代码
- 发现问题后直接修改
- 告诉 AI 修改原因，帮助其学习

### 11. 利用聊天历史

使用聊天历史来迭代之前的提示，避免重新开始。

**技巧**：
- 在对话中引用之前的上下文
- 使用 "基于之前的修改，再优化..." 这样的表述
- 保持对话的连续性

### 12. 有目的地选择模型

根据需要选择不同模型：

| 模型 | 适用场景 |
|------|----------|
| **Gemini** | 适合精确任务，如代码补全、简单重构 |
| **Claude** | 适合广泛应用，如架构设计、复杂逻辑 |
| **GPT-4** | 适合通用编程任务 |

## 四、快捷键速查表

| 快捷键 | 功能 |
|--------|------|
| `Tab` | 接受代码补全建议 |
| `Esc` | 拒绝代码补全建议 |
| `Ctrl/Cmd + →` | 逐字接受建议 |
| `Ctrl/Cmd + L` | 打开 AI 对话面板 |
| `Ctrl/Cmd + K` | 打开行内编辑对话框 |
| `Ctrl/Cmd + I` | 唤起 AI 助手（Composer） |
| `Ctrl/Cmd + /` | 快速切换 AI 模型 |

## 五、总结

将 Cursor 视为一个强大的初级开发者——如果你指明方向，它可以快速有效地工作。成功的关键在于：

1. **结构化工作流程**：分步骤、分文件处理任务
2. **有效控制**：明确规则、限定范围、审查输出
3. **持续迭代**：利用测试驱动、聊天历史不断优化
