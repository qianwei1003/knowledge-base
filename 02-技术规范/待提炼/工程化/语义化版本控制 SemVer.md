# 语义化版本控制规范 (SemVer 2.0.0)

> 来源：SemVer 官方 https://semver.org/lang/zh-CN/
>
> 作者：Tom Preston-Werner（Gravatar 创办人、GitHub 共同创办人）

## 版本格式

```
主版本号.次版本号.修订号
MAJOR.MINOR.PATCH
```

| 版本段 | 何时递增 | 对应变更 |
|--------|---------|---------|
| **MAJOR** | 做了不兼容的 API 修改 | Breaking Changes |
| **MINOR** | 做了向下兼容的功能性新增 | New Features |
| **PATCH** | 做了向下兼容的问题修正 | Bug Fixes |

先行版本号和版本编译信息可附加在后面：`1.0.0-alpha`、`1.0.0+20130313144700`

## 核心规则

1. **必须定义公共 API** — 在代码或文档中精确定义
2. **版本号格式** `X.Y.Z`，非负整数，禁止补零
3. **发行后禁止修改** — 任何修改必须发行新版本
4. **0.x.y = 开发阶段** — 公共 API 不稳定
5. **1.0.0 = API 成型** — 标志公共 API 已定义

### 递增规则

| 操作 | 何时 | 效果 |
|------|------|------|
| PATCH `x.y.Z` | 仅修复 Bug（不改变 API） | Z++, 其他归零 |
| MINOR `x.Y.z` | 新增功能（向下兼容）或标记弃用 | Y++, Z=0 |
| MAJOR `X.y.z` | 不兼容的 API 修改 | X++, Y=0, Z=0 |

### 先行版本号（Pre-release）

格式：`-` + 以 `.` 分隔的标识符
- 标识符：ASCII 字母数字和 `-`，禁止补零
- 示例：`1.0.0-alpha`、`1.0.0-alpha.1`、`1.0.0-0.3.7`
- 优先级**低于**标准版本

### 版本编译信息（Build Metadata）

格式：`+` + 以 `.` 分隔的标识符
- 示例：`1.0.0-alpha+001`、`1.0.0+20130313144700`
- 判断版本优先层级时**被忽略**

### 优先级排序

```
1.0.0-alpha < 1.0.0-alpha.1 < 1.0.0-alpha.beta
< 1.0.0-beta < 1.0.0-beta.2 < 1.0.0-beta.11
< 1.0.0-rc.1 < 1.0.0
```

## 与约定式提交的关系

| 约定式提交类型 | 对应 SemVer |
|:---:|:---:|
| `fix:` | PATCH |
| `feat:` | MINOR |
| `BREAKING CHANGE:` | MAJOR |

## FAQ 摘要

- **0.y.z 阶段**：从 0.1.0 开始，每次发布递增 Y
- **何时发布 1.0.0**：软件用于正式环境、API 稳定
- **误发不兼容版本**：立即发新次版本来修正并恢复向下兼容
- **弃用功能**：更新文档 + 在新的 MINOR 版本中标记弃用信息

## 正则表达式

```regex
^(0|[1-9]\d*)\.(0|[1-9]\d*)\.(0|[1-9]\d*)(?:-((?:0|[1-9]\d*|\d*[a-zA-Z-][0-9a-zA-Z-]*)(?:\.(?:0|[1-9]\d*|\d*[a-zA-Z-][0-9a-zA-Z-]*))*))?(?:\+([0-9a-zA-Z-]+(?:\.[0-9a-zA-Z-]+)*))?$
```
