# 百度前端编码规范 — JavaScript 篇

> 百度 EFE（前端工程部）官方 JavaScript 编码规范，国内最早、最系统化的前端编码规范之一。覆盖代码风格、语言特性、浏览器环境三大板块，以 [强制]/[建议] 分级标注规则重要性。

## 1 前言

本文档目标是使 JavaScript 代码风格保持一致，容易被理解和被维护。虽然本文档针对 JavaScript 设计，但在使用 TypeScript 等预编译语言时，适用的部分也应尽量遵循。

## 2 代码风格

### 2.1 文件

- **[建议]** JavaScript 文件使用无 BOM 的 UTF-8 编码
- **[建议]** 在文件结尾处，保留一个空行

### 2.2 结构

#### 2.2.1 缩进

- **[强制]** 使用 **4 个空格** 做为一个缩进层级，不允许使用 2 个空格或 tab 字符
- **[强制]** switch 下的 case 和 default 必须增加一个缩进层级

```javascript
// good
switch (variable) {
    case '1':
        // do...
        break;
    case '2':
        // do...
        break;
    default:
        // do...
}
```

#### 2.2.2 空格

| 规则 | 说明 |
|------|------|
| **[强制]** | 二元运算符两侧必须有一个空格，一元运算符与操作对象之间不允许有空格 |
| **[强制]** | 左花括号 `{` 前必须有一个空格 |
| **[强制]** | `if/else/for/while/function/switch/do/try/catch/finally` 关键字后必须有一个空格 |
| **[强制]** | 对象属性中 `:` 之后必须有空格，之前不允许有空格 |
| **[强制]** | 函数名和 `(` 之间不允许有空格 |
| **[强制]** | `,` 和 `;` 前不允许有空格；不在行尾时后必须跟一个空格 |
| **[强制]** | `()` 和 `[]` 内紧贴括号部分不允许有空格 |
| **[强制]** | 单行声明的数组与对象，`{}` 和 `[]` 内不允许包含空格 |
| **[强制]** | 行尾不得有多余的空格 |

#### 2.2.3 换行

- **[强制]** 每个独立语句结束后必须换行
- **[强制]** 每行不得超过 **120** 个字符
- **[强制]** 运算符处换行时，运算符必须在新行的行首
- **[强制]** 不允许在 `,` 或 `;` 前换行
- **[建议]** 对于 if...else、try...catch...finally 等语句，推荐在 `}` 后添加换行

#### 2.2.4 语句

- **[强制]** 不得省略语句结束的分号
- **[强制]** if/else/for/do/while 中即使只有一行，也不得省略块 `{...}`
- **[强制]** 函数定义结束不允许添加分号
- **[强制]** IIFE 必须在函数表达式外添加 `(`

### 2.3 命名

| 类型 | 命名法 | 示例 |
|------|--------|------|
| 变量 | Camel | `myName` |
| 常量 | 全大写+下划线 | `MAX_COUNT` |
| 函数 | Camel | `getUserName` |
| 类 | Pascal | `PersonModel` |
| 类的方法/属性 | Camel | `getName` |
| 枚举变量 | Pascal | `Direction` |
| 枚举属性 | 全大写+下划线 | `UP, DOWN` |
| 命名空间 | Camel | `utils` |

其他命名规则：
- **[强制]** 缩写词（如 XML、HTTP）根据命名法保持一致：`parseXML`（Camel）、`HTTPRequest`（Pascal）
- **[强制]** 类名使用名词
- **[建议]** 函数名使用动宾短语
- **[建议]** boolean 变量使用 `is` 或 `has` 开头
- **[建议]** Promise 对象用动宾短语的进行时

### 2.4 注释

- **[强制]** 单行注释必须独占一行，`//` 后跟一个空格
- **[建议]** 避免使用 `/*...*/` 多行注释，多用单行注释
- **[强制]** 文档化注释使用 `/**...*/`，前必须空一行
- **[建议]** 自文档化注释说明 **what**，而不是 **how**
- **[强制]** 类型定义 `{string}`, `{number}`, `{boolean}` 首字母必须小写
- **[强制]** 文件顶部必须包含 `@file` 标识
- **[强制]** 函数注释必须包含说明、参数和返回值类型

## 3 语言特性

### 3.1 变量

- **[强制]** 变量、函数在使用前必须先定义
- **[强制]** 每个 `var` 只能声明一个变量（禁止 `var a = 1, b = 2;`）
- **[强制]** 变量必须即用即声明，不得在函数起始位置统一声明

### 3.2 条件

- **[强制]** 使用严格等于 `===`，仅判断 null/undefined 时允许 `== null`
- **[建议]** 按执行频率排列分支顺序
- **[建议]** 相同变量的多值条件用 `switch` 代替 `if`
- **[建议]** 如果 `else` 块后无语句，可删除 `else`

### 3.3 循环

- **[建议]** 不在循环体中包含函数表达式，事先提取到循环体外
- **[建议]** 循环内多次使用的不变值在循环外用变量缓存
- **[建议]** 遍历有序集合时缓存 `length`
- **[建议]** 顺序无关的遍历使用逆序遍历

### 3.4 类型

#### 类型检测

- **[建议]** 优先使用 `typeof`
- **[建议]** 对象类型检测使用 `instanceof`
- **[建议]** null/undefined 检测使用 `== null`

#### 类型转换

| 目标类型 | 推荐方式 |
|---------|---------|
| string | `+ ''` |
| number | `+` 或 `Number()` |
| string→number（忽略尾部非数字） | `parseInt` |
| boolean | `!!` |
| 去除小数 | `Math.floor / Math.round / Math.ceil` |

- **[强制]** 使用 `parseInt` 时必须指定进制

### 3.5 字符串

- **[强制]** 字符串使用**单引号** `'`
- **[建议]** 使用数组或 `+` 拼接字符串
- **[建议]** HTML 拼接需合理转义
- **[建议]** 复杂数据到视图的转换使用模板引擎

### 3.6 对象

- **[强制]** 使用对象字面量 `{}` 创建新 Object
- **[强制]** 不允许修改原生对象和宿主对象的原型
- **[建议]** 属性访问尽量使用 `.`
- **[建议]** `for in` 遍历时使用 `hasOwnProperty` 过滤原型属性

### 3.7 数组

- **[强制]** 使用数组字面量 `[]` 创建新数组
- **[强制]** 遍历数组不使用 `for in`
- **[建议]** 清空数组使用 `.length = 0`

### 3.8 函数

- **[建议]** 函数长度控制在 **50 行** 以内
- **[建议]** 函数参数控制在 **6 个** 以内
- **[建议]** 通过 options 对象传递非数据输入型参数
- **[建议]** 适当时候将闭包内大对象置为 `null`
- **[建议]** 使用 IIFE 避免 Lift 效应

### 3.9 面向对象

- **[强制]** 类继承实现时需要修正 `constructor`
- **[建议]** 属性在构造函数中声明，方法在原型中声明
- **[强制]** 自定义事件名必须全小写
- **[强制]** 自定义事件只能有一个 `event` 参数

### 3.10 动态特性

- **[强制]** 避免使用直接 `eval`
- **[建议]** 尽量不要使用 `with`
- **[建议]** 减少 `delete` 的使用
- **[建议]** 避免修改外部传入的对象

## 4 浏览器环境

### 4.1 模块化（AMD）

- **[强制]** 使用 AMD 作为模块定义
- **[建议]** 定义模块时不要指明 `id` 和 `dependencies`
- **[强制]** 全局环境中 `require` 必须以 async 形式调用
- **[强制]** 内部模块 `require` 必须使用 relative id

### 4.2 DOM

#### 元素获取

- **[建议]** 单个元素优先使用 `document.getElementById`
- **[建议]** 多个元素使用 `context.getElementsByTagName`
- **[建议]** 获取直接子元素使用 `children`，避免 `childNodes`

#### 样式

- **[建议]** 获取实际样式使用 `getComputedStyle`
- **[建议]** 通过添加 `className` 改变样式，避免直接操作 `style`
- **[强制]** 设置带单位的非 0 值属性时，不允许省略单位

#### DOM 操作

- **[建议]** 尽量减少页面 reflow
- **[建议]** 尽量减少 DOM 操作

#### DOM 事件

- **[建议]** 优先使用 `addEventListener`，避免 HTML 属性绑定
- **[建议]** 适当时候移除事件监听器（防止内存泄漏）

## 总结

百度 JS 编码规范是国内前端规范体系的标杆，其核心特色：

1. **分级标注** — [强制] 和 [建议] 两级，团队可灵活执行
2. **全面覆盖** — 从代码风格到语言特性到浏览器环境
3. **实用性** — 每条规则都有"解释"和"示例"（good/bad）
4. **前瞻性** — 明确指出也适用于 TypeScript 等预编译语言
