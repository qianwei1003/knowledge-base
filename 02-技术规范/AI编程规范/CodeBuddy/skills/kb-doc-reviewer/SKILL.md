# 知识库文档规范审查工具

## 🎯 功能概述
审查现有文档是否符合知识库规范，生成详细的审查报告。

## 🔍 审查维度

### 维度1：文件位置审查
| 检查项 | 合规标准 | 权重 |
|:---|:---|:---|
| 目录合规性 | 文档位于正确的目录下 | 15% |
| Inbox流程 | 新文档是否经过Inbox流程 | 5% |

### 维度2：命名规范审查
| 检查项 | 合规标准 | 权重 |
|:---|:---|:---|
| 日期格式 | 使用YYYY-MM-DD格式 | 10% |
| 命名结构 | 符合"日期-主题.md"格式 | 15% |
| 主题明确性 | 主题简洁明确 | 5% |
| AI协作标识 | AI文档以"AI-"开头 | 5% |

### 维度3：元信息头审查
| 检查项 | 合规标准 | 权重 |
|:---|:---|:---|
| created字段 | 必填，格式正确 | 10% |
| source字段 | 必填，有效来源标识 | 10% |
| tags字段 | 必填，正确分类标签 | 10% |
| 完整性 | 无缺失必填字段 | 5% |

### 维度4：内容规范审查
| 检查项 | 合规标准 | 权重 |
|:---|:---|:---|
| 中文标点 | 使用中文标点符号 | 5% |
| 双链链接 | 包含相关`[[链接]]` | 5% |
| 结构层次 | 合理的标题层级 | 5% |

**总分计算**：`∑(检查项得分 × 权重)`，满分100分

## 🔧 审查流程

1. **读取文档** → 获取文档内容和元数据
2. **逐项检查** → 按四个维度进行检查
3. **评分计算** → 计算综合得分
4. **生成报告** → 输出详细审查报告
5. **提供建议** → 给出具体修复建议

## 🎯 审查标准示例

### 命名规范示例
- **合规**：`2026-04-18-会议记录.md`
- **不合规**：`会议记录.txt`、`0418会议.md`

### 元信息头示例
```yaml
# 合规
created: 2026-04-18
source: #来源/Claude
tags: [#技术/AI编程, #工作区/对话]

# 不合规
created: April 18, 2026  # 格式错误
source: Claude           # 缺少#来源/前缀
# 缺少tags字段
```
