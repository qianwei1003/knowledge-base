# 知识库文档规范写入工具

## 🎯 功能概述
指导用户按照知识库规范创建新文档，提供模板、位置推荐和MOC更新指导。

## 📋 核心检查项

### 1. 位置选择检查
| 文档类型 | 推荐位置 | 命名前缀 |
|:---|:---|:---|
| AI协作产出 | `06-AI协作区/` 或 `02-技术规范/` | `AI-YYYY-MM-DD-` |
| 技术规范 | `02-技术规范/` | 无特定前缀 |
| 工作记录 | `01-工作区/` | `YYYY-MM-DD-` |
| 项目文档 | `03-项目库/{项目名}/` | 项目相关 |
| 学习笔记 | `04-学习研究/` | 主题相关 |
| 新内容 | `01-工作区/00-Inbox/` | 临时命名 |

### 2. 命名规范检查
- ✅ `YYYY-MM-DD-主题.md` 或 `AI-YYYY-MM-DD-主题.md`
- ✅ 使用连字符`-`而非下划线`_`
- ✅ 主题简洁明确（不超过20字）
- ❌ 避免特殊字符和空格
- ❌ 避免纯数字或无意义名称

### 3. 元信息头检查（必须字段）
```yaml
created: YYYY-MM-DD
source: #来源/Claude  #来源/Copilot  #来源/User
tags: [#分类/子类, #工作区/类型]
```

**AI协作文档额外字段：**
```yaml
project: 项目名称
related: [["相关文档", "说明"]]
```

### 4. 内容质量检查
- ✅ 使用中文标点（，。；！？）
- ✅ 添加双链 `[[相关文档]]`
- ✅ 包含足够上下文
- ✅ 结构化组织（标题层次）
- ❌ 避免纯代码转储
- ❌ 避免敏感信息

### 5. MOC更新检查
1. **定位相关MOC**：查找文档所在目录的MOC文件
2. **添加索引项**：在MOC的适当位置添加新文档链接
3. **更新日期**：修改MOC的最后更新日期
4. **验证链接**：确保所有`[[链接]]`都能正确解析

## 🔧 使用流程

1. **确定文档类型** → 选择对应模板
2. **检查命名规范** → 使用正确命名格式
3. **填写元信息** → 确保必填字段完整
4. **编写内容** → 遵循质量要求
5. **更新MOC** → 添加到相关索引
