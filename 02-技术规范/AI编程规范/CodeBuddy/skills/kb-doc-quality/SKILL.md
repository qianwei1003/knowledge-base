# 知识库文档质量检查工具

## 🎯 功能概述
检查文档的歧义、错误、可读性等质量问题，提供优化建议。

## 🔍 质量检查维度

### 维度1：歧义性检查
| 检查项 | 说明 | 检查方法 |
|:---|:---|:---|
| 代词指代 | "这个"、"那个"指代是否明确 | 上下文分析 |
| 模糊表述 | "可能"、"大概"等模糊词语 | 关键词匹配 |
| 术语一致性 | 同一概念是否使用相同术语 | 术语表对比 |
| 指代清晰度 | 章节引用是否明确 | 链接检查 |

### 维度2：错误检查
| 检查项 | 说明 | 检查方法 |
|:---|:---|:---|
| 事实错误 | 技术事实是否准确 | 常识验证 |
| 逻辑矛盾 | 前后内容是否矛盾 | 逻辑推理 |
| 过期信息 | 信息是否已过时 | 日期检查 |
| 链接失效 | `[[链接]]`是否有效 | 文件存在性检查 |

### 维度3：可读性检查
| 检查项 | 说明 | 检查方法 |
|:---|:---|:---|
| 句子长度 | 长句是否过多 | 句子分割分析 |
| 段落结构 | 段落是否过长 | 段落长度统计 |
| 标题层次 | 标题层级是否合理 | 标题结构分析 |
| 中文表达 | 是否符合中文表达习惯 | 语言模式分析 |

### 维度4：完整性检查
| 检查项 | 说明 | 检查方法 |
|:---|:---|:---|
| 上下文缺失 | 是否缺少必要背景 | 上下文分析 |
| 步骤不完整 | 操作步骤是否完整 | 步骤序列检查 |
| 示例缺失 | 复杂概念是否有示例 | 示例代码/文字检查 |
| 结论缺失 | 分析是否有明确结论 | 结尾部分检查 |

## 🔧 检查流程

1. **文档解析** → 提取文档内容和结构
2. **多维度检查** → 并行检查四个维度
3. **问题分类** → 将问题归类到对应维度
4. **评分计算** → 计算综合质量分
5. **生成报告** → 输出详细质量报告
6. **优化排序** → 按优先级排序优化建议

## 🎯 检查标准示例

### 歧义性示例
- **问题**："这个功能需要优化"（未指明哪个功能）
- **建议**："用户登录功能需要优化响应时间"

### 可读性示例
- **问题**：连续5个句子超过40字的长句
- **建议**：将长句拆分为2-3个短句

### 完整性示例
- **问题**："按照以下步骤操作"但未列出步骤
- **建议**：添加编号步骤列表：1. ... 2. ...
