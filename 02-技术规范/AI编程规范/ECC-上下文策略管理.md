## 何时激活

- 运行接近上下文限制（200K+ tokens）的长会话
- 正在处理多阶段任务（研究 → 规划 → 实现 → 测试）
- 在同一会话中切换不相关任务
- 完成重大里程碑后开始新工作
- 当响应变慢或连贯性下降时（上下文压力）

## 为什么需要策略性压缩

**自动压缩的问题：**
- 通常在任意点触发
- 经常打断正在进行的任务
- 无法感知逻辑任务边界
- 可能中断复杂多步骤操作

**策略性压缩的优势：**

| 时机 | 压缩？ | 原因 |
|------|--------|------|
| 研究 → 规划 | 是 | 研究上下文庞大；规划是提炼后的输出 |
| 规划 → 实现 | 是 | 规划在 TodoWrite 或文件中；释放上下文给代码 |
| 实现 → 测试 | 可能 | 如果测试引用了近期代码则保留；切换关注则压缩 |
| 调试 → 下一特性 | 是 | 调试追踪会污染无关工作的上下文 |
| 实现中途 | 否 | 丢失变量名、文件路径和部分状态代价高昂 |
| 失败方案之后 | 是 | 在尝试新方法前清除死路推理 |

## 什么在压缩后保留

| 保留 | 丢失 |
|------|------|
| CLAUDE.md 指令 | 中间推理和分析 |
| TodoWrite 任务列表 | 之前读取的文件内容 |
| Memory 文件 (`~/.claude/memory/`) | 多步骤对话上下文 |
| Git 状态（commits、branches） | 工具调用历史和计数 |
| 磁盘上的文件 | 口头陈述的微妙用户偏好 |

## 最佳实践

1. **规划后压缩** — 一旦计划在 TodoWrite 中定稿，压缩后全新开始
2. **调试后压缩** — 为继续工作清理错误解决上下文
3. **不要在实现中途压缩** — 为相关变更保留上下文
4. **阅读建议** — 钩子告诉你何时，你自己决定是否
5. **压缩前写入** — 重要上下文保存到文件或 memory 后再压缩
6. **带摘要压缩** — 添加自定义消息：`/compact Focus on implementing auth middleware next`

---

**记住**：压缩的时机和内容与压缩本身同样重要。在逻辑边界处压缩，保留工作上下文。
