# 知识提炼规范 v2.0（简化版）

> v1.0 太重（4步流程+8字段frontmatter），62篇走完需要40+小时
> v2.0 简化为2步流程，每篇5-15分钟

---

## 流程

```
去重合并 → 提炼入库
(5min)      (5-15min)
```

### Step 1: 去重合并

- 同主题多文件 → 保留最全的1篇，其余删除
- 独特内容追加到保留文件的"附录"章节
- 每次合并记录：删除了什么、保留了什么

### Step 2: 提炼入库

对每篇待提炼文件执行：

1. **精简 frontmatter**（只保留2个必填字段）：
   ```yaml
   ---
   来源: [官方文档/博客/GitHub]
   原始链接: [URL]
   ---
   ```

2. **开头加核心要点**（3-5条 bullet，非表格）：
   ```markdown
   ## 核心要点
   - 规则1：描述
   - 规则2：描述
   - 规则3：描述
   ```

3. **删冗余保精华**：
   - 保留：规则 + ✅示例 + ❌反模式
   - 删除：铺垫文字、重复说明、过度背景

4. **移到正式目录**：提炼完成 → 从 `待提炼/` 移到对应正式分区

---

## 判断标准

| 文档质量 | 处理方式 |
|---------|---------|
| ⭐⭐⭐⭐⭐ 官方文档 | 优先提炼 |
| ⭐⭐⭐⭐ 大厂实践 | 正常提炼 |
| ⭐⭐⭐ 社区文章 | 选择性提炼 |
| ⭐⭐ 及以下 | 删除 |

**去重优先于收集**——已有同类主题就不新增。
