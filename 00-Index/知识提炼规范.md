# 知识提炼规范 v1.0

> 本规范定义如何将原始资料转化为结构化知识的标准流程，确保提炼结果的一致性、可追溯性和实用性。

---

## 一、目标定义

### 1.1 提炼目的

```
原始资料（混乱、冗长、多源） → 提炼知识（结构化、精简、统一） → 规则文件（可执行、可验证）
```

### 1.2 提炼层级

| 层级 | 名称 | 目标 | 产出位置 | 审核要求 |
|------|------|------|---------|---------|
| L1 | 初提炼 | 提取核心要点，建立结构 | `待提炼/` | 自审 |
| L2 | 精提炼 | 整合多源，消除冲突 | `06-AI协作区/提炼知识/` | 交叉审核 |
| L3 | 规则化 | 转换为可执行规则 | `rules/` | 终审发布 |

### 1.3 质量目标

- ✅ **结构化**：使用统一模板，便于机器解析
- ✅ **可执行**：规则描述清晰，可直接用于代码审查
- ✅ **有来源**：每个规则都能追溯到具体信息来源
- ✅ **可验证**：包含检查清单或测试用例
- ✅ **无冲突**：同一领域的规则保持一致

---

## 二、执行步骤

### 2.1 提炼流程图

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   1.预筛选   │ → │   2.初提炼   │ → │   3.精提炼   │ → │   4.规则化   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
       │                  │                  │                  │
       ▼                  ▼                  ▼                  ▼
  质量评估打分        提取核心要点        多源整合对比        转换为规则
  决定提炼价值        标记来源链接        消除矛盾冲突        编写检查清单
```

### 2.2 步骤详解

#### Step 1: 预筛选（5分钟/文档）

**目标**：判断文档是否值得提炼

**执行动作**：
1. 阅读文档 frontmatter 和开头部分
2. 评估质量等级（见质量标准章节）
3. 判断与现有知识库的重合度
4. 做出决策：
   - ✅ 值得提炼 → 进入 Step 2
   - ⏸️ 暂时搁置 → 移动到 `待提炼/低优先级/`
   - ❌ 无需提炼 → 删除或归档

**决策标准**：
| 条件 | 决策 |
|------|------|
| 质量 ≥ ⭐⭐⭐⭐ | 必须提炼 |
| 质量 ⭐⭐⭐ 且 内容独特 | 值得提炼 |
| 质量 ≤ ⭐⭐ 或 内容重复 | 搁置或删除 |

---

#### Step 2: 初提炼（15-30分钟/文档）

**目标**：提取核心要点，保留原始结构

**执行动作**：
1. **更新 frontmatter**
   ```yaml
   ---
   来源: [具体来源]
   领域: [分类]
   关键词: [标签列表]
   质量: ⭐⭐⭐⭐
   提炼状态: 初提炼完成
   提炼日期: 2026-04-19
   提炼人: Claude
   原始链接: [URL]
   ---
   ```

2. **添加核心要点速查**
   - 在文档开头添加 `## 核心要点速查`
   - 用表格形式列出 5-10 条最关键的规则
   - 每条规则标注来源段落

3. **标注关键段落**
   - 使用 `> 来源：xxx` 标记关键论述
   - 保留原始章节结构
   - 删除明显冗余的内容

**初提炼检查清单**：
- [ ] frontmatter 信息完整
- [ ] 核心要点速查已添加
- [ ] 关键段落已标注来源
- [ ] 冗余内容已删除
- [ ] 提炼状态已更新

---

#### Step 3: 精提炼（30-60分钟/主题）

**目标**：整合多源资料，形成统一知识体系

**执行动作**：
1. **主题归类**
   - 将相同主题的初提炼文档归类
   - 创建或更新 `06-AI协作区/提炼知识/` 下的主题文档

2. **多源整合**
   - 对比不同来源的相似规则
   - 标注冲突和差异
   - 形成统一的最佳实践

3. **结构化输出**
   - 使用标准模板（见模板章节）
   - 添加信息来源追溯表
   - 编写检查清单

**精提炼检查清单**：
- [ ] 相同主题文档已归类
- [ ] 多来源冲突已标注
- [ ] 统一规则已形成
- [ ] 使用标准模板
- [ ] 来源追溯表已添加
- [ ] 检查清单已编写

---

#### Step 4: 规则化（60-120分钟/规则文件）

**目标**：将知识转换为可执行的规则文件

**执行动作**：
1. **映射到规则目录**
   - 确定规则文件位置（`rules/common/` 或 `rules/{语言}/`）
   - 检查现有规则，避免重复

2. **编写规则内容**
   - 使用规则文件模板
   - 每条规则包含：描述、示例、检查方法
   - 添加 "规则来源" 注释

3. **验证可执行性**
   - 规则是否可用工具检查？
   - 规则是否容易理解？
   - 规则是否无歧义？

**规则化检查清单**：
- [ ] 规则文件位置正确
- [ ] 无重复规则
- [ ] 每条规则有示例
- [ ] 每条规则有检查方法
- [ ] 规则来源已标注
- [ ] 通过可执行性验证

### 3.2 质量标准

#### 星级评定标准

| 星级 | 标准 | 处理建议 |
|------|------|---------|
| ⭐⭐⭐⭐⭐ | 内容权威、结构清晰、示例丰富、可直接转化为规则 | 优先提炼 |
| ⭐⭐⭐⭐ | 内容较完整、有一定权威性、少量冗余 | 正常提炼 |
| ⭐⭐⭐ | 内容一般、部分有价值、需要大量整理 | 选择性提炼 |
| ⭐⭐ | 内容较少、价值有限 | 合并提炼或搁置 |
| ⭐ | 内容质量差、来源不可靠 | 删除 |

## 四、质量控制

### 4.1 审核流程

```
初提炼完成 ──→ 自审检查 ──→ [通过] ──→ 移动到 06-AI协作区/待精提炼/
                  │
                  └─→ [不通过] ──→ 返回修改

精提炼完成 ──→ 交叉审核 ──→ [通过] ──→ 移动到 06-AI协作区/提炼知识/
                  │
                  └─→ [不通过] ──→ 返回修改

规则化完成 ──→ 终审发布 ──→ [通过] ──→ 发布到 rules/
                  │
                  └─→ [不通过] ──→ 返回修改
```

### 4.2 审核检查清单

#### 初提炼审核清单

| 检查项 | 标准 | 检查方法 |
|--------|------|---------|
| frontmatter 完整 | 6个字段齐全 | 人工检查 |
| 核心要点速查 | 5-10条关键规则 | 人工检查 |
| 来源标注 | 关键段落有来源标记 | 搜索 `> 来源：` |
| 格式规范 | 使用正确 Markdown 语法 | 预览检查 |
| 无敏感信息 | 无密码、密钥等 | 人工检查 |

#### 精提炼审核清单

| 检查项 | 标准 | 检查方法 |
|--------|------|---------|
| 主题归类正确 | 与同主题文档放一起 | 人工检查 |
| 多源对比表 | 冲突和差异已标注 | 搜索 `多源对比` |
| 统一规则 | 无自相矛盾 | 人工阅读 |
| 使用标准模板 | 符合精提炼模板 | 人工检查 |
| 检查清单可执行 | 每条都能验证 | 人工评估 |

#### 规则化审核清单

| 检查项 | 标准 | 检查方法 |
|--------|------|---------|
| 位置正确 | 在正确的 rules/ 子目录 | 人工检查 |
| 无重复 | 不与现有规则重复 | 搜索关键词 |
| 每条规则有示例 | 正例+反例 | 人工检查 |
| 每条规则可检查 | 有明确的检查方法 | 人工评估 |
| 来源标注 | 有 `规则来源` 注释 | 搜索 `规则来源` |

### 4.3 质量指标

建议定期统计以下指标：

| 指标 | 计算方法 | 目标值 |
|------|---------|--------|
| 提炼完成率 | 已完成提炼数 / 待提炼总数 | > 80% |
| 平均提炼时间 | 总提炼时间 / 提炼文档数 | < 30分钟/文档 |
| 返工率 | 返工次数 / 提交审核次数 | < 10% |
| 规则转化率 | 转化为规则的提炼文档 / 精提炼文档 | > 30% |

## 六、常见问题

### Q: 如何判断一个文档是否值得提炼？
**A:** 使用预筛选步骤的决策标准。主要看：内容质量（星级）、与现有知识库的重合度、内容独特性。当不确定时，可以先做初提炼，在过程中判断价值。

### Q: 多个来源的规则冲突怎么办？
**A:** 在精提炼阶段，创建"多源对比"表格，标注冲突点。然后：
1. 优先采用更权威的来源（如官方文档 > 社区文章）
2. 如果权威性相当，优先采用更严格/更具体的规则
3. 在规则中标注"此规则存在争议，推荐做法为..."

### Q: 提炼过程中发现原文档有错误怎么办？
**A:** 
1. 在提炼文档中标注错误，并说明正确的做法
2. 保留原文的错误示例，但标注 `❌ 原文错误，正确做法见...`
3. 不修改原始文档（除非你有权限且确定需要修改）

### Q: 提炼后的文档应该放在哪里？
**A:** 
- 初提炼完成：`02-技术规范/待提炼/{分类}/`
- 精提炼完成：`06-AI协作区/提炼知识/`
- 规则化完成：`rules/{common|语言}/`

### Q: 如何处理更新频繁的来源文档？
**A:** 
1. 在 frontmatter 中记录版本号和更新日期
2. 提炼时关注核心原则（变化慢）而非具体实现（变化快）
3. 定期（如每季度）检查来源文档是否有重大更新
