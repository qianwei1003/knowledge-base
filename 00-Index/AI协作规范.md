## ✍️ AI 写入规则

### 1. 文件命名规范
```
AI-{YYYY-MM-DD}-{主题}.md      # AI 协作产出
AI-{YYYY-MM-DD}-{项目}-{类型}  # 分类更细的产出
```

**示例：**
- `AI-2026-04-18-代码审查-React组件.md`
- `AI-2026-04-18-技术方案-微服务架构.md`

### 2. 必须包含的元信息
```markdown
---
created: 2026-04-18
source: #来源/Claude
project: codebuddy-craft
tags: [#技术/AI编程, #工作区/对话]
related: [["相关笔记名", "链接说明"]]
---

# 内容标题
```

### 3. 内容质量要求
- ✅ 包含足够的上下文信息
- ✅ 使用中文标点和格式
- ✅ 添加相关的双链 `[[链接]]`
- ✅ 标注信息来源和依据
- ❌ 不包含纯代码转储（应引用文件路径）
- ❌ 不包含敏感信息

### 4. 写入时机

**Claude Code 应主动写入的情况：**
- 完成重要的代码规范制定
- 完成技术方案设计
- 解决复杂问题后的经验总结
- 会议讨论的重要结论
- 项目进度的重要里程碑

**Claude Code 可以写入的情况：**
- 用户明确要求"记录到知识库"
- 对话中产生的可复用知识
- 重要的决策及其理由

### 5. 写入流程
```
1. 确定写入位置 → 2. 检查命名规范 → 3. 写入内容
        ↓
4. 添加元信息 → 5. 建立双链 → 6. 更新 MOC
```

### 6. 冲突处理
- **文件已存在**：创建新版本 `AI-xxx-v2.md` 而非覆盖
- **内容重复**：在现有文件中追加而非新建
- **命名冲突**：在文件名后加序号
