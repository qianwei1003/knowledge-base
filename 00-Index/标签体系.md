# 标签体系

> 统一的标签规范，确保知识库内容可被有效检索和关联。

## 🏷️ 标签结构

```
#来源/xxx        # 内容来源
#区域/xxx        # 地理/组织区域
#领域/xxx        # 专业领域
#项目/xxx        # 具体项目
#技术/xxx        # 技术分类
#工作区/xxx      # 工作流程分类
#学习/xxx        # 学习相关
#个人/xxx        # 个人相关
#状态/xxx        # 内容状态
#优先级/xxx      # 重要程度
```

## ⚠️ 注意事项

1. **层级限制**：建议最多 2 级（如 `#技术/前端`，避免 `#技术/前端/React/17`）
2. **中文优先**：中文内容使用中文标签
3. **统一命名**：相同概念使用同一标签，避免 `#React` 和 `#react` 混用
4. **定期清理**：定期合并重复标签，删除无用标签

## 🔄 标签维护

- **新建标签**：确保符合本体系，避免随意创建
- **合并标签**：发现重复时使用 Tag Wrangler 合并
- **归档标签**：不用的标签移动到 `tags-归档/` 注释掉
