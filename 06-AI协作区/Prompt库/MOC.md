## 4. Prompt 设计原则

### 4.1 基础结构
```
角色设定 + 任务描述 + 输出格式 + 约束条件 + 示例（可选）
```

### 4.2 质量标准
- ✅ 角色清晰（你是xxx）
- ✅ 任务具体（请做xxx）
- ✅ 格式明确（以xxx格式输出）
- ✅ 约束完整（不要xxx，必须xxx）
- ❌ 模糊指令（帮我写点代码）
- ❌ 缺少边界（随便写）

### 4.3 迭代方法
1. 记录初始 Prompt 和输出
2. 标注不满意的地方
3. 针对性修改 Prompt
4. 对比效果，保留最优版本
