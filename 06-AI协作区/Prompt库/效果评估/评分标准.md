# Prompt 效果评分标准

> 统一的 Prompt 效果评估标准，确保评分客观、可比较。

---

## 一、评分维度

| 维度 | 说明 | 权重 |
|------|------|------|
| 代码质量 | 代码是否规范、可读、可维护 | 25% |
| 规范遵守 | 是否遵守 Prompt 中的约束条件 | 25% |
| 可维护性 | 输出是否易于后续修改和扩展 | 20% |
| 可执行性 | 输出是否可直接使用，无需大改 | 20% |
| 效率 | 是否减少了人工工作量 | 10% |

---

## 二、评分标准

### ⭐（1分）- 不可用
- 输出内容与需求严重不符
- 存在关键性错误
- 无法直接使用，需要完全重写

### ⭐⭐（2分）- 需大量修改
- 基本结构正确，但细节问题多
- 需要手动修改 50% 以上内容
- 有一定的参考价值

### ⭐⭐⭐（3分）- 可用但需调整
- 输出基本正确
- 需要小范围修改（10-30%）
- 可以直接使用但效果一般

### ⭐⭐⭐⭐（4分）- 良好
- 输出质量较高
- 只需微调（<10%）
- 符合预期，可直接使用

### ⭐⭐⭐⭐⭐（5分）- 优秀
- 输出完全符合预期或超出预期
- 无需修改或极少修改
- 可作为标准示例复用

---

## 三、评估流程

```
1. 执行 Prompt → 2. 检查输出 → 3. 按维度打分 → 4. 计算加权平均 → 5. 记录结果
```

### 评估检查清单

| 检查项 | 是否满足 |
|--------|---------|
| 输出格式是否符合要求 | ✅ / ❌ |
| 所有约束条件是否遵守 | ✅ / ❌ |
| 是否有语法/逻辑错误 | ✅ / ❌ |
| 是否缺少关键内容 | ✅ / ❌ |
| 是否有冗余内容 | ✅ / ❌ |
