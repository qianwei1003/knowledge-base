# 代码阅读理解

## 适用场景

- 快速理解陌生代码库
- 阅读他人写的代码
- 理解复杂逻辑的实现

## Prompt内容

```
你是一个资深的代码审查专家。

请帮我理解以下代码：

【代码内容】
{代码内容}

【我需要理解的内容】
1. 这段代码的功能是什么？
2. 核心逻辑的执行流程是什么？
3. 关键变量/函数的作用是什么？
4. 有哪些潜在的边界情况？
5. 代码的优缺点是什么？

【输出格式】
## 功能概述
（一句话概括代码功能）

## 核心逻辑流程
（用步骤或流程图说明）

## 关键部分解释
- 变量/函数名：作用说明

## 边界情况
- 情况1：如何处理
- 情况2：如何处理

## 代码评价
- 优点：
- 可改进点：
```

## 使用示例

**输入：**
```
{代码内容}
export function debounce<T extends (...args: any[]) => any>(
  fn: T,
  delay: number
): (...args: Parameters<T>) => void {
  let timer: ReturnType<typeof setTimeout> | null = null;

  return function (this: any, ...args: Parameters<T>) {
    if (timer) clearTimeout(timer);
    timer = setTimeout(() => {
      fn.apply(this, args);
    }, delay);
  };
}
```

**输出：**
```markdown
## 功能概述
防抖函数：在事件触发后延迟执行，若延迟期间再次触发则重新计时。

## 核心逻辑流程
1. 调用 debounce(fn, delay) 返回一个新函数
2. 每次调用新函数时：
   - 清除之前的定时器（如果有）
   - 设置新的定时器，delay 毫秒后执行原函数
3. 只有"最后一次"触发会在 delay 毫秒后真正执行

## 关键部分解释
- `fn: T`：要防抖的原始函数
- `delay: number`：延迟时间（毫秒）
- `timer`：定时器引用，用于取消之前的延迟执行
- `this: any`：保持原函数的 this 指向
- `Parameters<T>`：保留原函数的参数类型

## 边界情况
- 短时间内多次触发：只有最后一次会执行
- delay = 0：立即.reset Additional 异步执行
- fn 内部抛错：需要在 fn 调用处加 try-catch（当前代码未处理）

## 代码评价
优点：
- TypeScript 类型完整，保留原函数类型
- 正确处理 this 指向
- 泛型设计，适用于任意函数

可改进点：
- 缺少错误处理
- 缺少取消防抖的能力
- 可添加 immediate 选项（首次立即执行）

## 使用示例
const handleSearch = debounce((query: string) => {
  fetchResults(query);
}, 300);

input.addEventListener('input', (e) => {
  handleSearch(e.target.value);
});
```
