# 测试用例生成

## 适用场景

- 为现有代码生成单元测试
- 提高测试覆盖率
- 验证代码行为符合预期

## Prompt内容

```
你是一个精通软件测试的 QA 工程师。

请为以下代码生成单元测试：

【代码文件】
{代码内容或文件路径}

【测试框架】
- 框架：{Jest / Vitest / Mocha}
- 是否需要 DOM 测试：{是 / 否}

【约束条件】
1. 测试用例覆盖：正常情况、边界情况、异常情况
2. 使用 describe/it 结构组织测试
3. 测试描述清晰，说明测试什么
4. 使用 mock 隔离外部依赖
5. 每个 it 只测试一个行为
6. 包含 beforeEach/afterEach 清理环境

【输出格式】
1. 测试文件完整代码
2. 覆盖场景列表（说明测试了哪些场景）
3. 建议补充的测试场景（如果有）
```

## 使用示例

**输入：**
```
{代码内容}
function formatDate(date: Date, format: string = 'YYYY-MM-DD'): string {
  if (!(date instanceof Date) || isNaN(date.getTime())) {
    throw new Error('Invalid date');
  }

  const pad = (n: number) => n.toString().padStart(2, '0');

  return format
    .replace('YYYY', date.getFullYear().toString())
    .replace('MM', pad(date.getMonth() + 1))
    .replace('DD', pad(date.getDate()));
}

{框架}
Jest
{DOM测试}
否
```

**输出：**
```typescript
// formatDate.test.ts
import { formatDate } from './formatDate';

describe('formatDate', () => {
  describe('正常情况', () => {
    it('应使用默认格式 YYYY-MM-DD 格式化日期', () => {
      const date = new Date('2024-01-15');
      expect(formatDate(date)).toBe('2024-01-15');
    });

    it('应正确处理月份补零', () => {
      const date = new Date('2024-03-05');
      expect(formatDate(date)).toBe('2024-03-05');
    });

    it('应正确处理日期补零', () => {
      const date = new Date('2024-12-01');
      expect(formatDate(date)).toBe('2024-12-01');
    });
  });

  describe('边界情况', () => {
    it('应正确处理年末日期', () => {
      const date = new Date('2024-12-31');
      expect(formatDate(date)).toBe('2024-12-31');
    });

    it('应正确处理年初日期', () => {
      const date = new Date('2024-01-01');
      expect(formatDate(date)).toBe('2024-01-01');
    });
  });

  describe('异常情况', () => {
    it('应在传入无效 Date 对象时抛出错误', () => {
      const invalidDate = new Date('invalid');
      expect(() => formatDate(invalidDate)).toThrow('Invalid date');
    });

    it('应在传入非 Date 对象时抛出错误', () => {
      expect(() => formatDate('2024-01-01' as any)).toThrow('Invalid date');
    });

    it('应在传入 null 时抛出错误', () => {
      expect(() => formatDate(null as any)).toThrow('Invalid date');
    });
  });
});

// 覆盖场景列表
// - 默认格式化
// - 月份补零
// - 日期补零
// - 年末边界
// - 年初边界
// - 无效日期对象
// - 非日期类型
// - null 值

// 建议补充的测试场景：
// - 闰年日期
// - 自定义格式参数
// - 其他分隔符格式
```

## 效果评估

| 维度 | 评分 | 说明 |
|------|------|------|
| 覆盖完整性 | ⭐⭐⭐⭐⭐ | 正常/边界/异常都覆盖 |
| 可维护性 | ⭐⭐⭐⭐⭐ | 结构清晰，描述明确 |
| 可执行性 | ⭐⭐⭐⭐⭐ | 可直接运行 |
| 代码质量 | ⭐⭐⭐⭐ | 符合测试规范 |

**优点：**
- 三种场景（正常/边界/异常）都覆盖
- 测试描述清晰
- 自动给出覆盖场景列表

**缺点：**
- 复杂异步逻辑可能需要手动调整
- Mock 设置可能不完整
