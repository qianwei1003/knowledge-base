# AI工具技能配置经验

## 背景

用户希望让多个 AI 工具（CodeBuddy、WorkBuddy、QClaw）能够访问同一个个人知识库，形成统一的智能体知识库访问体系。

## 核心方案

创建统一的 `personal-knowledge-base` skill，让所有 AI 工具通过相同接口访问知识库。

## 踩坑记录

### 1. QClaw Skill 格式（重要）

QClaw/OpenClaw 的 skill 本质上是一个**目录 + SKILL.md**：

```
~/.agents/skills/<skill-name>/
└── SKILL.md              ← 必需的核心文件
```

**SKILL.md 结构：**
```yaml
---
name: skill-name
description: 技能描述
metadata:
  openclaw:
    user-invocable: true   # 用户可直接触发
---

# 技能名称

## 详细指令...
```

**关键配置项：**
| 字段 | 作用 |
|------|------|
| `name` | Skill 目录名和标识 |
| `description` | 触发词和功能描述 |
| `user-invocable` | 是否暴露为用户可触发命令 |
| `disable-model-invocation` | 是否禁止模型自动调用 |

### 2. Skill 放置位置

QClaw 自动扫描的 skills 目录：
- `~/.agents/skills/` ← 新建技能放这里
- `~/.openclaw/skills/` ← OpenClaw 格式
- `<workspace>/skills/` ← 工作区技能
