# AI-2026-04-18-AI编程工具配置文件研究

## 研究背景

需要为 CodeBuddy 和 qclaw 配置个人知识库访问能力，调研了各 AI 编程工具的配置文件位置和配置方法。

---

## 核心发现

### 配置文件类型对比

| 格式 | 适用范围 | 维护方 |
|------|----------|--------|
| **CLAUDE.md** | Claude Code 专用 | Anthropic |
| **AGENTS.md** | 60+ 种工具通用 | Linux Foundation (Agentic AI Foundation) |

### 层级结构（由低到高）

| 优先级 | 路径 | 说明 |
|--------|------|------|
| 1 | `~/.claude/CLAUDE.md` | 全局配置（适用于所有项目） |
| 2 | `./CLAUDE.md` | 项目根目录配置 |
| 3 | 子目录 CLAUDE.md | 按需加载 |
| 4 | 路径限定规则 (YAML frontmatter) | glob 匹配激活 |

> **低层级是对高层级的补充而非替换**

---

## 各工具配置路径

### Claude Code / CodeBuddy

```
~/.claude/CLAUDE.md          # 全局配置
./CLAUDE.md                  # 项目根目录
```

**关键信息：**
- Claude Code 自动读取 `~/.claude/CLAUDE.md`
- 全局配置中的指令会自动应用于所有项目
- 建议篇幅保持在 300 行以内

### OpenAI Codex

```
~/.codex/AGENTS.md           # 全局配置
./AGENTS.md                  # 项目根目录
```

**限制：** 有 32 KiB 的大小限制

### 其他工具

| 工具 | 全局配置路径 |
|------|-------------|
| GitHub Copilot | `~/.github/copilot-instructions.md` |
| Windsurf | `~/.codeium/windsurf/memories/` (自动记忆) |
| Cursor | `.cursor/rules/*.mdc` (非全局) |

## 配置方法总结

### 方案一：修改全局 CLAUDE.md（已实施）

路径：`~/.claude/CLAUDE.md`

已在文件中添加知识库信息：

```markdown
## 📚 Personal Knowledge Base

Your human has a personal knowledge base at `C:\Users\60597\Desktop\知识库`.

**When to search it:**
- User asks about past work, notes, or documents
- User references previous decisions or projects
...

**How to access:**
Use `read_file` tool with path like `C:\Users\60597\Desktop\知识库\{category}\{filename}.md`
```

### 方案二：qclaw AGENTS.md（已实施）

路径：`~/.qclaw/workspace/AGENTS.md`

已在 Memory 部分之后添加知识库章节。

## 最佳实践

### ✅ 应包含

- 无法从代码推断的 Bash 命令
- 与语言默认配置不同的代码风格规则
- 精确的测试/构建命令（含参数）
- 知识库位置和访问方法
- 项目工作流约定

### ❌ 应排除

- Agent 阅读代码就能明白的内容
- 标准语言规范
- 详细 API 文档（改为提供链接）
- 通用建议（如"写简洁代码"）
- 频繁变动的信息
