# codebuddy-craft 案例

> 企业级编程规范助手，帮助用户搭建 rule/skill/agent 规范体系。

---

## 项目概述

### 目标
帮助用户创建企业级 AI 编程规范体系：
- 生成 Rule 文件（.cursorrules、AGENTS.md、copilot-instructions.md）
- 搭建 Skill（可复用的能力模块）
- 设计 Agent（特定场景的 AI 助手）
- 提供编程工作相关的知识库

### 平台
- 主要平台：CodeBuddy（腾讯 AI 编程助手）
- 兼容平台：Cursor、Claude Code、GitHub Copilot

## 核心能力

### 1. 规范生成能力
- 支持生成 .cursorrules、AGENTS.md、copilot-instructions.md
- 基于知识库自动填充规范内容
- 支持自定义规范模板

### 2. 知识库管理
- 五大领域知识库：前端、后端、DevOps、安全、测试
- 支持持续更新和扩展
- 与 Obsidian 集成，便于维护

### 3. 创建记录
- 记录每次规范创建的详细信息
- 支持回溯和迭代

## 交互流程

```
用户：帮我创建一个前端开发项目的规范

Agent：
1. 识别场景 → 前端开发
2. 加载知识库 → frontend.md + 规范收集
3. 提问确认 → 技术栈？团队规模？重点要求？
4. 生成规范 → 输出 .cursorrules / AGENTS.md
5. 记录创建 → logs/creation-log.md

用户：帮我创建一个 React + TypeScript 项目的规范

Agent：
1. 识别场景 → 前端开发 + React + TypeScript
2. 加载知识库 → frontend.md + React最佳实践
3. 生成规范 → 包含 React 组件规范、TypeScript 类型规范等
```

## 技术亮点

### 1. 知识库与规则分离
- 知识库是原材料
- 规则是加工后的产品
- 两者分离，便于维护

### 2. 多平台兼容
- 生成规则适配多个 AI 编程工具
- 核心知识库通用

### 3. Obsidian 集成
- 知识库用 Obsidian 管理
- 支持双链、标签、MOC
- 便于长期积累

## 经验总结

### 成功经验
| 经验 | 说明 |
|------|------|
| 先设计再动手 | 明确目标用户、核心功能、交互流程 |
| 知识库分层 | Sources（原始）→ Knowledge（提炼）→ Output（产物） |
| 记录创建过程 | 便于复盘和迭代 |

### 遇到的坑
| 问题 | 解决方案 |
|------|---------|
| 命名规范打架 | 统一为一套命名规范 |
| 元信息格式不一致 | 统一为一种格式 |
| 目录路径写错 | 及时修正，建立校验机制 |

### 可改进点
| 改进项 | 计划 |
|--------|------|
| 自动化测试 | 添加规范文件的验证脚本 |
| 知识库同步 | 与大厂规范源保持同步 |
| 模板扩展 | 增加更多场景的模板 |
