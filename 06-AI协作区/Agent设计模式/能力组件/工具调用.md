# 工具调用

## 概述

工具调用是 Agent 执行外部操作的核心能力，使 Agent 能够与外部系统交互、获取实时信息、执行具体任务。

---

## 工具类型

### 1. 信息获取类
| 工具 | 作用 | 示例 |
|------|------|------|
| 搜索 | 搜索互联网信息 | web_search |
| 查询 | 查询数据库/API | api_query |
| 读取 | 读取文件/网页 | file_read, web_fetch |

### 2. 操作执行类
| 工具 | 作用 | 示例 |
|------|------|------|
| 写入 | 写入文件/数据库 | file_write |
| 调用 | 调用外部API | api_call |
| 执行 | 执行代码/命令 | code_exec |

### 3. 辅助能力类
| 工具 | 作用 | 示例 |
|------|------|------|
| 计算 | 数学计算 | calculator |
| 翻译 | 语言翻译 | translator |
| 格式化 | 格式转换 | formatter |

## 工具调用流程

```
1. Agent 分析任务
      ↓
2. 选择合适的工具
      ↓
3. 构造工具参数
      ↓
4. 执行工具调用
      ↓
5. 获取返回结果
      ↓
6. 解析并整合结果
      ↓
7. 继续推理或返回用户
```

## 关键设计

### 1. 工具选择
- 基于工具描述匹配任务需求
- 多个工具可完成同一任务时，按优先级选择
- 可配置工具白名单/黑名单

### 2. 参数验证
```python
def validate_params(tool_name, params):
    schema = get_tool_schema(tool_name)
    # 检查必填参数
    for required in schema.get("required", []):
        if required not in params:
            raise ValueError(f"缺少必填参数: {required}")
    # 检查参数类型
    for key, value in params.items():
        expected_type = schema["properties"][key]["type"]
        if not isinstance(value, expected_type):
            raise TypeError(f"参数 {key} 类型错误")
    return True
```

### 3. 错误处理
| 错误类型 | 处理方式 |
|---------|---------|
| 工具不存在 | 返回错误，提示可用工具 |
| 参数错误 | 返回错误，提示正确格式 |
| 执行超时 | 重试或降级处理 |
| 权限不足 | 返回错误，提示权限需求 |

### 4. 结果处理
- 结构化结果：直接使用
- 非结构化结果：解析提取
- 错误结果：传递给Agent处理

## 安全控制

### 1. 权限分级
| 级别 | 工具示例 | 控制方式 |
|------|---------|---------|
| 低风险 | 搜索、计算 | 自动允许 |
| 中风险 | 文件读取 | 需确认 |
| 高风险 | 文件写入、命令执行 | 必须确认 |

### 2. 沙箱隔离
- 代码执行在沙箱环境中
- 限制文件访问范围
- 限制网络访问白名单

## 最佳实践

### 1. 工具设计原则
- 单一职责：一个工具只做一件事
- 描述清晰：让Agent能正确理解和使用
- 参数精简：只保留必要参数
- 返回规范：统一返回格式

### 2. 工具组合
```
搜索工具 → 读取工具 → 分析工具 → 输出
（搜索找到链接）→（读取详细内容）→（分析内容）→（生成答案）
```

### 3. 容错设计
- 工具不可用时有降级方案
- 关键工具调用失败时通知用户
- 提供工具状态监控
