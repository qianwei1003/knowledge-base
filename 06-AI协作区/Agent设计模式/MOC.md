## 2. Agent 设计原则

### 2.1 核心原则
1. **目标清晰**：Agent 必须有明确的职责边界
2. **能力适配**：能力配置匹配目标，不多不少
3. **可控可干预**：关键节点允许人工干预
4. **可追溯**：记录决策过程，便于复盘
5. **渐进式**：从简单开始，逐步增强能力

### 2.2 反模式（避免）
- ❌ 万能 Agent（什么都能做 = 什么都做不好）
- ❌ 黑盒决策（无法解释为什么这样做）
- ❌ 无限记忆（上下文爆炸）
- ❌ 过度自动化（该人工确认的不确认）

## 5. 设计流程

```
1. 明确目标 → 2. 选择架构 → 3. 配置能力 → 4. 设计交互 → 5. 测试迭代
```

### 各阶段要点
| 阶段 | 关键问题 |
|------|---------|
| 明确目标 | Agent 要解决什么问题？成功的标准是什么？ |
| 选择架构 | 单Agent 还是多Agent？需要人机协作吗？ |
| 配置能力 | 需要哪些工具？记忆要多长？ |
| 设计交互 | 用户如何触发？如何干预？输出格式是什么？ |
| 测试迭代 | 典型场景测试 → 发现问题 → 调整设计 |
