# 第一性原理

## 核心概念

**回归事物本质，从最基础的事实出发思考，而不是类比或假设。**

> "与其比较，不如拆解。" —— 亚里士多德

---

## 详细解释

### 起源
亚里士多德最早提出：每个系统中存在第一性原理，是最基础的命题，不能被省略或删除。

### 与类比思维的区别

| 思维方式 | 做法 | 例子 |
|---------|------|------|
| 类比思维 | 看别人怎么做，照着做 | "竞品有这个功能，我们也加" |
| 第一性原理 | 拆解到最基本元素，重新组合 | "用户真正需要的是什么？我们如何用最优方式实现？" |

### Elon Musk 的应用
造火箭时，Musk 发现买现成火箭太贵。
- 类比思维：大家都买，我也买，贵是必然
- 第一性原理：火箭材料成本只占总成本2%，问题在哪？重新设计生产流程，SpaceX 把成本降到 1/10

---

## 使用方法

### 步骤
1. **识别问题**：明确要解决什么
2. **拆解问题**：分解到最基础的事实 / 元素
3. **质疑假设**：哪些"理所当然"其实可以被改变？
4. **重新组合**：从基础元素出发，寻找新方案

### 实践模板
```
我想解决：[问题]
传统做法是：[类比]
但这背后的基础事实是：
  1. [事实1]
  2. [事实2]
  3. [事实3]
如果不受传统做法限制，我可以用这些事实这样重新组合：
  → [新方案]
```

## 应用场景

### 场景1：产品设计
- 问题：用户流失率高
- 类比：竞品有积分体系，我们也加？
- 第一性原理：
  - 用户为什么流失？→ 核心需求没被满足
  - 核心需求是什么？→ 某功能缺失、体验差、没有场景
  - 解决方案：先补核心功能，再考虑留存

### 场景2：职业选择
- 问题：应该跳槽吗？
- 类比：同龄人都在跳，薪资涨幅20%，我也跳？
- 第一性原理：
  - 我工作的目标是什么？→ 技能成长 + 收入
  - 当前岗位能否达成目标？→ 能 / 不能
  - 新岗位能否更好？→ 能 / 不能
  - 结论：基于事实判断，而不是看别人怎么做

### 场景3：知识库设计
- 问题：知识库应该怎么组织？
- 类比：别人用 PARA，我也用 PARA
- 第一性原理：
  - 知识库的目的是什么？→ 快速找到信息、积累知识、产出价值
  - 我的现状是什么？→ 有哪些内容、习惯是什么
  - 组织方式应该匹配我的习惯，而不是照搬

## 局限性

| 局限 | 说明 |
|------|------|
| 耗时 | 拆解需要时间，紧急问题可能来不及 |
| 信息不足时难用 | 没有足够的事实，拆解不出来 |
| 过度拆解风险 | 拆得太细可能失去全局视角 |

**适用：** 重要决策、复杂问题、创新设计
**不适用：** 紧急问题、信息不足、日常小事
