## Function Calling 工具定义

### 核心概念
Function Calling（函数调用）是让 LLM 能够使用外部工具的核心机制：
1. 模型决定何时调用工具
2. 选择调用哪个工具
3. 生成符合格式的调用参数
4. 开发者执行后返回结果

### 调用流程（5步）
```
用户输入 → LLM识别需求 → 生成调用请求 → 执行函数 → 返回结果 → LLM整合回答
```

### 工具定义示例

```json
{
  "name": "get_stock_quote",
  "description": "获取股票实时行情数据",
  "parameters": {
    "type": "object",
    "properties": {
      "ts_code": {
        "type": "string",
        "description": "股票代码，如 000001.SZ"
      }
    },
    "required": ["ts_code"]
  }
}
```

## Agent Plan 机制

### ReAct vs Function Calling

| 模式 | 特点 | 适用场景 |
|------|------|----------|
| ReAct | 文本提示词驱动 | 简单任务 |
| Function Calling | 结构化API调用 | 复杂工具调用 |

### 核心要素
1. **Planner**: 任务分解与规划
2. **Executor**: 工具调用执行
3. **Observer**: 结果观察与判断
4. **Loop Control**: 循环终止条件
