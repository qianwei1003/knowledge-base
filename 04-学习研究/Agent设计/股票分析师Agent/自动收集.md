## Function Calling 工具定义

### 核心概念
Function Calling（函数调用）是让 LLM 能够使用外部工具的核心机制：
1. 模型决定何时调用工具
2. 选择调用哪个工具
3. 生成符合格式的调用参数
4. 开发者执行后返回结果

### 调用流程（5步）
```
用户输入 → LLM识别需求 → 生成调用请求 → 执行函数 → 返回结果 → LLM整合回答
```

### 工具定义示例

```json
{
  "name": "get_stock_quote",
  "description": "获取股票实时行情数据",
  "parameters": {
    "type": "object",
    "properties": {
      "ts_code": {
        "type": "string",
        "description": "股票代码，如 000001.SZ"
      }
    },
    "required": ["ts_code"]
  }
}
```

### ReAct vs Function Calling

| 模式 | 特点 | 适用场景 |
|------|------|----------|
| ReAct | 文本提示词驱动 | 简单任务 |
| Function Calling | 结构化API调用 | 复杂工具调用 |

### 核心要素
1. **Planner**: 任务分解与规划
2. **Executor**: 工具调用执行

| 分析师 | 基本面分析师(FA) | 财务数据分析、估值评估 | 股票代码、日期范围 | PE/PB/ROE、估值报告 |
| 分析师 | 市场分析师(MA) | 技术分析、趋势判断 | 股票代码、技术指标列表 | 趋势分析、支撑阻力位 |
| 分析师 | 新闻分析师(NA) | 新闻情绪分析、事件识别 | 股票代码、新闻源 | 情绪评分、关键事件 |

| 研究员 | 看涨研究员(Bull) | 乐观角度挖掘机会 | 基本面+市场分析报告 | 看涨投资建议 |
| 研究员 | 看跌研究员(Bear) | 悲观角度识别风险 | 新闻+社交媒体报告 | 看跌风险警示 |
| 风控 | 激进评估(AGG) | 高风险高收益策略 | 综合报告 | 风险评分 |
| 风控 | 保守评估(CON) | 低风险稳健策略 | 综合报告 | 风险评分 |
| 风控 | 中性评估(NEU) | 平衡风险收益 | 综合报告 | 风险评分 |
| 决策 | 交易员(Trader) | 综合建议→交易决策 | 研究员辩论结果 | 买卖信号+仓位 |

### 辩论机制（关键设计）
- Bull/Bear 研究员最多 **3轮对抗性辩论**
- 目标：消除确证偏误（Confirmation Bias）和AI幻觉
- 最终由研究经理综合评估，避免单一视角偏差

## Function Calling 金融场景工具定义（完整版）

**来源**: https://www.runoob.com/ai-agent/ai-agent-function-calling.html

### 工具定义最佳实践

**优质工具描述的4个原则**：
| 原则 | 不好的示例 | 好的示例 |
|------|----------|---------|
| 描述具体 | "查询股票" | "获取指定A股股票实时行情，包括最新价、涨跌幅、成交量" |
| 参数详细 | `"symbol": {"type": "string"}` | `"symbol": {"type": "string", "description": "股票代码，如'000001.SZ'，6位数字+交易所后缀"}` |
| 枚举限制 | 自由文本输入 | `"enum": ["买入", "卖出", "观望"]` |
| 提供示例 | 无 | `"description": "日期格式'YYYY-MM-DD'，例如'2025-01-15'"` |
