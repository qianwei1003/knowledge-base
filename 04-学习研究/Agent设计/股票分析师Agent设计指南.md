## 二、工具定义设计

### 2.1 核心工具分类

```
┌─────────────────────────────────────────────────────────┐
│                   股票分析师 Agent 工具集                    │
├─────────────────────────────────────────────────────────┤
│  数据获取层                                                │
│  ├── 行情数据：实时行情、历史K线、分时数据                    │
│  ├── 财务数据：年报、季报、财务指标                         │
│  ├── 公告数据：公司公告、新闻资讯                           │
│  └── 宏观数据：GDP、CPI、利率、汇率                        │
├─────────────────────────────────────────────────────────┤
│  分析工具层                                                │
│  ├── 技术指标：MACD、KDJ、布林带、RSI                      │
│  ├── 形态识别：K线形态、趋势线、支撑阻力                     │
│  └── 估值分析：PE、PB、PS、DCF                            │
├─────────────────────────────────────────────────────────┤
│  工具调用层                                                │
│  ├── Function Calling / Tool Use                         │
│  └── ReAct 推理循环                                       │
└─────────────────────────────────────────────────────────┘
```

### 2.2 行情数据工具

| 工具名称 | 功能描述 | 数据源 |
|----------|----------|--------|
| `get_realtime_quote` | 获取实时行情 | 新浪/腾讯/iTick |
| `get_daily_kline` | 获取日K线数据 | AkShare/Tushare |
| `get_minute_data` | 获取分时分钟线 | iTick/Ashare |
| `get_realtime_trade` | 实时交易推送 | WebSocket |

**工具定义示例（Function Calling）**

```json
{
  "name": "get_realtime_quote",
  "description": "获取股票实时行情数据，包括当前价格、涨跌幅、成交量等",
  "parameters": {
    "type": "object",
    "properties": {
      "ts_code": {
        "type": "string",
        "description": "股票代码，格式：000001.SZ（上交所.SS，深交所.SZ）"
      }
    },
    "required": ["ts_code"]
  }
}
```

### 2.3 财务数据工具

| 工具名称 | 功能描述 |
|----------|----------|
| `get_financial_report` | 获取财务报表（利润表、资产负债表、现金流量表） |

| `get_annual_report` | 获取年度报告摘要 |

### 2.4 技术分析工具

| 工具名称 | 功能描述 |
|----------|----------|
| `calculate_macd` | 计算 MACD 指标 |
| `calculate_kdj` | 计算 KDJ 指标 |

## 四、Agent 架构设计

### 4.1 单 Agent 架构

适用于简单场景：查询、解释、分析

```
用户输入 → LLM + Prompt → 工具调用 → 结果整合 → 回答
```

**优点**：简单、响应快
**缺点**：复杂任务处理能力有限

### 4.2 多 Agent 架构（参考 TradingAgents）

适用于专业分析场景

```
┌─────────────┐
│  用户输入   │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  调度 Agent  │ ← 分析任务类型，分发到专业 Agent
└──────┬──────┘
       │
       ├──→ 基本面 Agent ──→ 财务分析
       ├──→ 技术面 Agent ──→ 指标分析
       ├──→ 情绪 Agent  ──→ 舆情分析
       └──→ 风控 Agent  ──→ 风险评估
       
       ▼
┌─────────────┐
│  综合 Agent  │ ← 整合各 Agent 结果
└──────┬──────┘
       │
       ▼
    最终报告
```

### 4.3 ReAct 推理模式

```
Thought（思考） → Action（行动） → Observation（观察） → ...
```

**示例**：
```
Thought: 用户想了解茅台的走势，我需要先获取实时行情
Action: call get_realtime_quote("600519.SH")
Observation: 当前价格 1850元，涨幅 2.3%，成交量放大

Thought: 需要进一步分析技术面，调用MACD指标
Action: call calculate_macd("600519.SH", period=20)
Observation: MACD金叉，DEA线上穿零轴

Thought: 综合分析...
```
